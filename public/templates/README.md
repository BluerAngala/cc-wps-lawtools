# 模板文件说明

## 目录结构

```
public/templates/
├── templates.json          # 模板配置文件
├── README.md              # 本说明文档
├── 民事起诉状.docx        # 模板文件（Word 格式）
├── 民事答辩状.docx
├── 代理词.docx
├── 劳动合同.docx
└── 律师函.docx
```

## 快速生成模板

使用自动化脚本一键生成所有模板：

```bash
pnpm create-templates
```

脚本会自动创建所有内置模板文档。

## 如何添加新模板

### 1. 创建模板文件

**方式一：使用脚本（推荐）**
- 编辑 `scripts/create-template-docs.js`
- 添加新模板定义
- 运行 `pnpm create-templates`

**方式二：手动创建**
- 在 Word 中创建新文档
- 添加内容和格式
- 保存为 `.docx` 格式到此目录

**重要说明：**
- 模板文件必须是 `.docx` 格式（Word 文档）
- 可以包含完整的格式、样式、表格等
- 支持变量占位符，格式：`{{变量名}}`

### 2. 更新配置文件

在 `templates.json` 中添加新模板的配置信息：

```json
{
  "templates": [
    {
      "id": "template_006",           // 唯一ID，格式：template_xxx
      "name": "模板名称",              // 显示名称
      "category": "诉讼文书",          // 分类：诉讼文书/合同模板/律师函/自定义
      "description": "模板描述",       // 简短描述
      "scene": "使用场景说明",         // 适用场景
      "fileName": "模板名称.docx",     // 文件名（必须与实际文件名一致）
      "isBuiltIn": true               // 是否为内置模板
    }
  ]
}
```

### 3. 配置说明

- **id**: 唯一标识符，建议使用 `template_` 前缀 + 数字编号
- **name**: 模板显示名称，会在界面上显示
- **category**: 模板分类，可选值：
  - `诉讼文书`: 起诉状、答辩状、代理词等
  - `合同模板`: 各类合同文书
  - `律师函`: 律师函相关
  - `自定义`: 其他类型
- **description**: 模板的简短描述，说明模板的用途
- **scene**: 详细的使用场景说明，帮助用户选择合适的模板
- **fileName**: 模板文件名，必须与实际文件名完全一致
- **isBuiltIn**: 标记为内置模板，内置模板不可删除

## 变量占位符规范

模板中使用 `{{变量名}}` 作为占位符，用户应用模板后可以手动替换这些占位符。

常用变量示例：
- `{{原告姓名}}`、`{{被告姓名}}`
- `{{出生日期}}`、`{{身份证号}}`
- `{{联系电话}}`、`{{住址}}`
- `{{法院名称}}`、`{{案由}}`
- `{{日期}}`

## 技术说明

### 模板加载机制

1. **配置加载**：启动时从 `templates.json` 读取模板配置
2. **按需加载**：模板内容在需要时才通过 WPS API 加载
3. **路径处理**：自动适配开发环境和生产环境的文件路径

### 应用模板流程

- **新建文档**：直接使用 WPS API 打开 docx 文件，保留所有格式
- **应用到当前文档**：读取模板内容后插入到当前文档

## 注意事项

1. 模板文件必须是 `.docx` 格式（Word 文档）
2. 文件名必须与配置文件中的 `fileName` 字段完全一致
3. 配置文件 `templates.json` 必须是有效的 JSON 格式
4. 添加新模板后，刷新页面即可看到新模板
5. 模板文件中的所有格式、样式都会被保留
6. 确保模板文件放在 `public/templates/` 目录下
7. 开发和生产环境都会从相对路径加载模板文件
