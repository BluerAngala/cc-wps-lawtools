# 调试窗格按钮点击无反应问题指南

## 问题描述
在 WPS 加载项的调试窗格中，点击按钮没有反应或功能不正常。

## 调试步骤

### 1. 基础环境检查

#### 1.1 打开浏览器开发者工具
- 在调试窗格中按 `F12` 打开开发者工具
- 切换到 `Console` 标签页

#### 1.2 检查是否有 JavaScript 错误
- 查看控制台是否有红色错误信息
- 如果有错误，记录错误信息以便进一步分析

#### 1.3 验证 WPS API 加载状态
在控制台中输入以下命令检查：
```javascript
// 检查 WPS Application 对象
console.log('WPS Application:', typeof window.Application)
console.log('当前活动文档:', !!window.Application?.ActiveDocument)
console.log('文档数量:', window.Application?.Documents?.Count)

// 检查 wps 全局对象
console.log('wps 全局对象:', typeof wps)
```

### 2. 测试按钮功能

#### 2.1 使用测试按钮
- 点击页面顶部的红色 "🔧 测试按钮点击" 按钮
- 如果弹出提示框，说明 Vue 组件响应正常
- 查看控制台输出的环境检查信息

#### 2.2 测试具体功能按钮
按以下顺序测试按钮：

1. **插入当前时间** - 最简单的功能，应该能正常工作
2. **取文件名** - 测试基础文档操作
3. **提取纯文本** - 测试文档内容读取
4. **添加页眉** - 测试文档修改功能

### 3. 常见问题及解决方案

#### 3.1 WPS Application 对象不存在
**症状**: 控制台显示 `window.Application` 为 `undefined`

**原因**: 
- 加载项未正确加载到 WPS 环境中
- 在浏览器中直接打开了页面，而不是在 WPS 中

**解决方案**:
1. 确保在 WPS 中打开文档
2. 通过 WPS 的加载项功能加载此插件
3. 检查 `manifest.xml` 配置是否正确

#### 3.2 ActiveDocument 为空
**症状**: `window.Application.ActiveDocument` 为 `null` 或 `undefined`

**原因**: 当前没有打开任何文档

**解决方案**:
1. 在 WPS 中新建或打开一个文档
2. 确保文档处于活动状态

#### 3.3 按钮点击有反应但功能不工作
**症状**: 控制台有调试信息输出，但具体功能失败

**原因**: 
- WPS API 调用失败
- 权限不足
- 文档状态不正确

**解决方案**:
1. 检查控制台的详细错误信息
2. 确保文档不是只读状态
3. 尝试启用修订模式

#### 3.4 AI 相关功能不工作
**症状**: AI 处理按钮点击后无反应或报错

**原因**:
- AI 服务配置问题
- 网络连接问题
- API 密钥配置错误

**解决方案**:
1. 检查环境变量配置
2. 确保网络连接正常
3. 验证 AI 服务 API 密钥

### 4. 调试信息收集

如果问题仍然存在，请收集以下信息：

#### 4.1 环境信息
```javascript
// 在控制台中运行
console.log('User Agent:', navigator.userAgent)
console.log('WPS 版本信息:', window.Application?.Version)
console.log('当前 URL:', window.location.href)
```

#### 4.2 错误日志
- 复制控制台中的所有错误信息
- 注意错误发生的时间和触发操作

#### 4.3 功能测试结果
记录每个按钮的测试结果：
- ✅ 正常工作
- ❌ 完全无反应
- ⚠️ 有反应但功能异常

### 5. 高级调试

#### 5.1 手动测试 WPS API
```javascript
// 测试基础 API
try {
  const doc = window.Application.ActiveDocument
  console.log('文档名称:', doc.Name)
  console.log('文档内容长度:', doc.Range().Text.length)
} catch (error) {
  console.error('API 调用失败:', error)
}
```

#### 5.2 测试任务窗格操作
```javascript
// 测试任务窗格
try {
  const taskPane = window.Application.GetTaskPane('your_taskpane_id')
  console.log('任务窗格状态:', taskPane?.Visible)
} catch (error) {
  console.error('任务窗格操作失败:', error)
}
```
