# 配置路径获取故障排查指南

## 🔍 问题：获取路径失败

当点击"查看配置文件路径"按钮时显示"获取路径失败"，说明遇到了环境问题。

---

## 📊 可能的情况

### 情况 1：在浏览器中调试（开发环境）

**显示信息：**
```
💾 浏览器 localStorage: wps_addon_config
```

**说明：**
- 这是**正常情况**
- 您当前在浏览器中通过 `npm run dev` 或 `wpsjs dbug` 运行插件
- 配置保存在浏览器的 localStorage 中
- 这不是错误！

**查看配置：**
1. 按 `F12` 打开浏览器开发者工具
2. 切换到 "Application" 或 "应用" 标签
3. 左侧找到 "Local Storage" → `http://localhost:3889`
4. 查找键名：`wps_addon_config`

---

### 情况 2：在 WPS 中运行

**正常显示：**
```
C:\Users\{用户名}\AppData\Roaming\Kingsoft\office6\wps_addon_config.json
```

**如果显示错误：**
```
❌ 获取失败: 无法获取 WPS 用户数据路径
```

**可能原因：**
1. WPS API 未正确加载
2. WPS 版本不支持该 API
3. 插件权限问题

---

## 🛠️ 解决方案

### 方案 1：确认当前环境

**在浏览器控制台中运行：**
```javascript
// 检查 WPS API 是否可用
console.log('WPS API 可用:', typeof window.Application !== 'undefined')

// 如果可用，获取路径
if (typeof window.Application !== 'undefined') {
  try {
    const path = window.Application.Env.GetUserDataPath()
    console.log('用户数据路径:', path)
    console.log('配置文件路径:', path + '\\wps_addon_config.json')
  } catch (e) {
    console.error('获取失败:', e)
  }
}
```

### 方案 2：手动查找配置文件

**步骤 1：打开文件资源管理器**
1. 按 `Win + R` 打开运行对话框
2. 输入以下路径之一并回车：

```
%APPDATA%\Kingsoft
```

或

```
%LOCALAPPDATA%\Kingsoft
```

**步骤 2：搜索配置文件**
- 在打开的文件夹中搜索：`wps_addon_config.json`
- 可能的位置：
  - `office6\`
  - `WPS Office\`
  - `addons\`

**步骤 3：常见路径**
```
C:\Users\{用户名}\AppData\Roaming\Kingsoft\office6\wps_addon_config.json
C:\Users\{用户名}\AppData\Local\Kingsoft\WPS Office\wps_addon_config.json
```

### 方案 3：重新生成配置文件

**如果找不到配置文件：**

1. **在设置页面修改任何配置**（如修改超时时间）
2. **配置会自动保存**
3. **再次点击"查看配置文件路径"**

配置文件会在首次保存时自动创建。

### 方案 4：使用 appConfig 工具

**在浏览器控制台运行：**
```javascript
// 导入配置管理器（如果在插件环境中）
import { appConfig } from './src/utils/appConfig.js'

// 获取配置路径
const path = appConfig.getConfigPath()
console.log('配置路径:', path)
```

## 📝 环境判断

### 开发环境（浏览器）

**特征：**
- ✅ 地址栏显示：`http://localhost:3889`
- ✅ 使用 `npm run dev` 或 `wpsjs dbug` 启动
- ✅ 配置保存在 localStorage

**验证方法：**
```javascript
console.log('当前环境:', window.location.href)
console.log('WPS API:', typeof window.Application !== 'undefined' ? '可用' : '不可用')
```

### 生产环境（WPS）

**特征：**
- ✅ 在 WPS 文档中运行
- ✅ 地址栏显示：`wps://...` 或内嵌页面
- ✅ 配置保存在文件系统

**验证方法：**
```javascript
if (typeof window.Application !== 'undefined') {
  console.log('WPS 版本:', window.Application.Version)
  console.log('用户数据路径:', window.Application.Env.GetUserDataPath())
}
```

## 🔧 调试技巧

### 技巧 1：查看完整错误信息

打开浏览器控制台（F12），点击"查看配置文件路径"按钮，查看控制台输出的详细错误信息。

### 技巧 2：检查 appConfig 状态

```javascript
// 在控制台运行
import { appConfig } from './src/utils/appConfig.js'

// 获取当前配置
const config = appConfig.getConfig()
console.log('当前配置:', config)

// 尝试保存配置
const success = appConfig.saveConfig(config)
console.log('保存结果:', success)
```

### 技巧 3：验证文件是否存在

如果获取到了路径，可以验证文件是否真的存在：

```javascript
// 仅在 WPS 环境中有效
if (typeof window.Application !== 'undefined') {
  const path = window.Application.Env.GetUserDataPath() + '\\wps_addon_config.json'
  const fso = new ActiveXObject('Scripting.FileSystemObject')
  console.log('配置文件存在:', fso.FileExists(path))
}
```

## ⚠️ 常见错误

### 错误 1：ActiveXObject 不可用

**错误信息：**
```
ReferenceError: ActiveXObject is not defined
```

**原因：**
- 在浏览器环境中运行（ActiveXObject 是 IE/WPS 特有）

**解决：**
- 这是正常的，配置会自动降级到 localStorage
- 在 WPS 环境中不会出现此错误

### 错误 2：权限被拒绝

**错误信息：**
```
Permission denied
```

**原因：**
- WPS 插件权限不足
- 文件路径没有写入权限

**解决：**
1. 以管理员身份运行 WPS
2. 检查 manifest.xml 中的权限配置
3. 联系 IT 检查文件夹权限

### 错误 3：路径包含特殊字符

**错误信息：**
```
Invalid path
```

**原因：**
- 用户名包含中文或特殊字符

**解决：**
- 通常不需要处理，WPS API 会自动处理编码
- 如果仍有问题，手动复制路径并在资源管理器中粘贴

在报告问题前，请确认以下事项：

- [ ] 确认当前运行环境（浏览器 or WPS）
- [ ] 查看浏览器控制台的错误信息
- [ ] 尝试手动搜索配置文件
- [ ] 尝试修改配置后再次查看路径
- [ ] 检查 WPS 版本是否支持相关 API

### 提供以下信息：

1. **运行环境**
   ```javascript
   console.log({
     url: window.location.href,

     wpsVersion: window.Application?.Version || '未知',

   ```

2. **错误信息**
   - 完整的控制台错误输出
   - 点击按钮后显示的具体信息

3. **配置内容**
   ```javascript
   // 如果在浏览器
   console.log(localStorage.getItem('wps_addon_config'))
   ```
