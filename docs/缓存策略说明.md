# 缓存策略说明

## 概述

本WPS插件现在实现了智能缓存管理策略，确保每次打开新文档时自动清除旧的缓存数据，避免不同文档间的数据混淆。

## 主要改进

### 1. 缓存时长调整
- **之前**: 24小时过期
- **现在**: 30分钟过期
- **原因**: 更短的缓存时间确保数据的时效性，避免长时间缓存过期数据

### 2. 文档切换监听
- **功能**: 自动检测WPS文档的打开、切换和关闭
- **触发条件**: 
  - 打开新文档
  - 切换到不同文档
  - 文档名称或路径发生变化
- **执行动作**: 自动清除所有AI分析缓存

### 3. 缓存状态显示
- **位置**: 合同审查界面的操作按钮下方
- **显示内容**: 
  - 当前缓存条目数量
  - 缓存过期时间设置
  - 文档监听状态

## 使用说明

### 自动缓存清除
1. **无需手动操作**: 每次打开新文档时自动清除
2. **监听频率**: 每秒检查一次文档变化
3. **清除范围**: 所有AI分析相关的缓存数据

### 手动缓存管理
1. **查看缓存状态**: 在合同审查界面查看实时缓存信息
2. **手动清除**: 点击"清除缓存"按钮
3. **缓存信息**: 显示当前缓存数量、过期时间和监听状态

### 缓存策略
1. **内容哈希**: 基于文档内容生成唯一哈希值
2. **分析类型**: 不同AI分析类型独立缓存
3. **双重存储**: 内存缓存 + 本地存储
4. **自动清理**: 定期清理过期缓存

## 优势

### 1. 数据准确性
- 避免不同文档间的缓存数据混淆
- 确保AI分析结果对应正确的文档
- 减少因缓存导致的错误结果

### 2. 性能优化
- 更短的缓存时间减少内存占用
- 自动清理机制保持系统性能
- 智能检测避免不必要的处理

### 3. 用户体验
- 透明的缓存管理，用户无感知
- 实时的缓存状态显示
- 可控的手动清除选项

## 配置选项

### 缓存管理器配置
```javascript
const cacheManager = new CacheManager({
  maxCacheSize: 500,           // 最大缓存条目数
  maxCacheAge: 30 * 60 * 1000, // 30分钟过期时间
  storagePrefix: 'contract_ai_cache_' // 存储前缀
})
```

### 文档监听器配置
```javascript
const documentWatcher = new DocumentWatcher(cacheManager)
// 检查间隔: 1000ms (1秒)
// 自动启动: 应用加载后2秒
```

## 故障排除

### 常见问题

1. **文档监听器未启动**
   - 检查WPS Application对象是否可用
   - 确认应用完全加载后再启动监听
   - 查看控制台是否有错误信息

2. **缓存未清除**
   - 检查文档是否真的发生了切换
   - 确认缓存管理器实例正确传递
   - 手动调用清除缓存功能

3. **缓存状态显示异常**
   - 检查全局缓存管理器是否正确初始化
   - 确认组件能访问到window.cacheManager
   - 刷新页面重新加载组件

- 当前文档信息
