# 关键词模式修复说明

## 问题描述

在关键词模式下点击"开始处理"按钮时，系统错误地将任务当作 AI 任务处理，执行了 `extractText` 操作来获取文档内容，而不是直接执行关键词批注/修订操作。

## 根本原因

在 `taskManager.js` 中：

1. `contractReview` 任务类型被添加到 `AI_RULE_TYPES` 数组中
2. 执行任务时，系统判断 `contractReview` 需要 AI 处理
3. 因此先调用 `extractText` 获取文档内容，准备发送给 AI
4. 但在关键词模式下，不需要 AI 处理，应该直接执行批注/修订

## 修复方案

### 1. 在 `taskManager.js` 中区分关键词模式和 AI 模式

**修改位置**: `src/services/contract/taskManager.js` 第 98-134 行

**核心逻辑**:
```javascript
// 特殊处理：关键词模式下的 contractReview 应该直接执行，不需要 AI
const isKeywordMode = params.mode === 'keyword'
const needsAI = this.AI_RULE_TYPES.includes(ruleType) && !isKeywordMode
```

**说明**:
- 检查 `params.mode` 是否为 `'keyword'`
- 如果是关键词模式，即使 `ruleType` 是 `contractReview`，也不走 AI 流程
- 直接调用 `executeDirectTask` 方法

### 2. 增强 `executeDirectTask` 方法处理关键词模式

**修改位置**: `src/services/contract/taskManager.js` 第 170-182 行

**核心逻辑**:
```javascript
case 'contractReview':
  // 关键词模式：直接调用 addComment（已支持批注和修订）
  if (params.mode === 'keyword') {
    return await taskPane.onbuttonclick('addComment', {
      keywordList: params.keywordList
    })
  } else {
    // AI 预审模式
    return await taskPane.onbuttonclick('contractReview', {
      reviewRules: params.reviewRules,
      aiResult: params.aiResult
    })
  }
```

**说明**:
- 在关键词模式下，直接调用 `addComment` 方法
- `addComment` 方法已增强，支持同时处理批注和修订
- AI 预审模式保持原有逻辑不变

### 3. 增强现有的 `addComment` 方法（支持修订）

**修改位置**: `src/services/wps/wpsTestHelper.js` 第 136-192 行

**功能增强**:
- ✅ 保留原有的批注功能
- ✅ 新增支持修订功能
- ✅ 统一方法，无需创建额外的函数
- ✅ 智能识别 `actionType`，自动选择批注或修订
- ✅ 统计并显示处理结果

**核心代码逻辑**:
```javascript
keywordList.forEach((item) => {
  const actionType = item?.actionType || '批注'
  const suggestedText = item?.suggestedText || ''
  
  if (actionType === '修订' && suggestedText) {
    // 修订：替换文本并添加批注说明
    selection.Text = suggestedText
    commentRange?.Comments?.Add(commentRange, comment || `已修订为"${suggestedText}"`)
    revisionCount++
  } else {
    // 批注：只添加批注
    commentRange?.Comments?.Add(commentRange, comment)
    commentCount++
  }
})
```

**设计优势**:
- 代码更简洁，只需维护一个方法
- 向后兼容，不影响现有功能
- 智能处理混合场景（批注+修订）

## 数据流程

### 修复前（错误流程）
```
用户点击"开始处理"（关键词模式）
  ↓
ContractServices.vue: executeSmartComment()
  ↓
contractService.executeTask('contractReview', config)
  ↓
taskManager.executeTask()
  ↓
判断：contractReview 在 AI_RULE_TYPES 中 ✓
  ↓
调用 extractText 获取文档内容 ← 【错误：不应该走这个流程】
  ↓
准备发送给 AI 处理...
```

### 修复后（正确流程）
```
用户点击"开始处理"（关键词模式）
  ↓
ContractServices.vue: executeSmartComment()
  ↓
contractService.executeTask('contractReview', { mode: 'keyword', keywordList: [...] })
  ↓
taskManager.executeTask()
  ↓
判断：isKeywordMode = true, needsAI = false
  ↓
executeDirectTask('contractReview', params)
  ↓
检查是否有修订类型
  ↓
调用 addCommentWithRevision() 或 addComment()
  ↓
直接在文档中添加批注/修订 ✓
```

## 关键词数据结构要求

### 批注类型
```javascript
{
  keyword: '关键词',
  comment: '批注内容',
  actionType: '批注'
}
```

### 修订类型
```javascript
{
  keyword: '关键词',
  comment: '修订说明（可选）',
  actionType: '修订',
  suggestedText: '建议的替换文本'  // 必填
}
```

**注意**: 
- 修订类型必须提供 `suggestedText`，否则会按批注处理
- `comment` 在修订模式下会添加到修订后的文本上，说明修订原因

### 测试场景 1：纯批注模式

]

**预期结果**: 找到"仲裁"后替换为"人民法院"，并添加批注说明

```javascript
keywordList: [
  { keyword: '第一条', comment: '需要确认', actionType: '批注' },
  { 
    keyword: '仲裁', 
    comment: '建议改为法院管辖', 
    actionType: '修订',
    suggestedText: '人民法院'
  }
]
```
**预期结果**: "第一条"添加批注，"仲裁"修订为"人民法院"并添加批注
