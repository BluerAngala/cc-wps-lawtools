# 应用配置管理说明

## 概述

项目已实现应用配置管理系统（`appConfig`），所有配置项集中管理，支持可视化设置、导入导出等功能。

## 配置结构

### 1. AI 服务配置 (`ai`)
- `apiKey`: AI API 密钥
- `baseUrl`: API 基础地址
- `model`: 使用的模型名称
- `timeout`: 请求超时时间（毫秒）

### 2. 金山文档配置 (`kdocs`)
- `webhookUrl`: Webhook 地址
- `token`: 访问令牌
- `sheetId`: 工作表 ID
- `apiUrl`: API 地址（可选）

### 3. 合同要素配置 (`extractor`)
- `extractTags`: 要提取的合同要素标签数组

### 4. 关键词批注配置 (`keyword`)
- `keywordList`: 关键词列表
  - `keyword`: 关键词
  - `comment`: 批注内容

### 5. 合同审查配置 (`review`)
- `contractReviewRules`: 审查规则列表
  - `reviewRules`: 审查规则名称
  - `reviewRequirements`: 审查要求描述
  - `actionType`: 操作类型（批注/高亮/标记）

### 6. 系统设置 (`system`)
- `firstLoadCompleted`: 首次加载是否完成
- `showWelcome`: 是否显示欢迎页面
- `autoSave`: 是否自动保存
- `theme`: 主题（暂未使用）

## 使用方式

### 在代码中使用

```javascript
import { appConfig } from '@/utils/appConfig.js'

// 获取完整配置
const config = appConfig.getConfig()

// 获取指定分类配置
const aiConfig = appConfig.get('ai')
const kdocsConfig = appConfig.get('kdocs')

// 设置指定分类配置
appConfig.set('ai', {
  apiKey: 'new-key',
  baseUrl: 'https://api.example.com',
  model: 'gpt-4',
  timeout: 60000
})

// 保存完整配置
appConfig.saveConfig(config)

// 重置配置
const defaultConfig = appConfig.reset()
```

### 在设置页面使用

1. 点击功能区的"设置"按钮
2. 在不同标签页中配置各项参数
3. 点击"保存配置"按钮保存
4. 支持导出/导入配置文件

## 配置优先级

配置读取优先级：**应用配置 > 环境变量 > 默认值**

1. 首先尝试从 `appConfig` 读取
2. 如果未配置，回退到 `.env` 文件中的环境变量
3. 最后使用代码中的默认值

## 配置持久化

- **主要方式**：使用 WPS 文件系统 API 存储
- **降级方案**：当 WPS API 不可用时，自动降级到 `localStorage`
- **存储位置**：`%UserDataPath%\wps_addon_config.json`
- **数据格式**：JSON

### 存储机制

1. **WPS 环境**：使用 `ActiveXObject('Scripting.FileSystemObject')` 将配置保存到用户数据目录
2. **浏览器环境**：自动降级到 `localStorage`，方便开发调试
3. **配置文件路径**：通过 `window.Application.Env.GetUserDataPath()` 获取

### 优势

- ✅ 配置持久化，不受浏览器清理影响
- ✅ 多文档共享配置
- ✅ 支持手动编辑配置文件
- ✅ 开发环境自动降级，无需 WPS 即可调试

## 配置导入导出

### 导出配置
```javascript
appConfig.exportConfig()
```
会下载一个 JSON 文件，包含当前所有配置。

### 导入配置
```javascript
const file = // 用户选择的文件
const config = await appConfig.importConfig(file)
```
从 JSON 文件恢复配置。

## 服务集成

### AI 服务 (`siliconflow.js`)
```javascript
import { appConfig } from '@/utils/appConfig.js'

const getAIConfig = () => {
  const config = appConfig.get('ai')
  return {
    apiKey: config.apiKey || import.meta.env.VITE_AI_API_KEY,
    baseUrl: config.baseUrl || import.meta.env.VITE_AI_API_BASE_URL,
    timeout: config.timeout || 30000
  }
}

// 重新初始化客户端（配置更新后调用）
import { reinitializeAIClient } from '@/services/ai/siliconflow.js'
reinitializeAIClient()
```

### 金山文档服务 (`kdocs.js`)
```javascript
import { appConfig } from '@/utils/appConfig.js'

const getKdocsConfig = () => {
  const config = appConfig.get('kdocs')
  return {
    webhookUrl: config.webhookUrl || import.meta.env.VITE_KDOCS_WEBHOOK_URL,
    token: config.token || import.meta.env.VITE_KDOCS_TOKEN,
    // ...
  }
}
```

## 迁移指南

### 从旧配置迁移

如果你的代码使用了旧的 `ConfigManager`：

**旧代码：**
```javascript
import { configManager } from '@/utils/configManager.js'
const config = configManager.loadConfig()
```

**新代码：**
```javascript
import { appConfig } from '@/utils/appConfig.js'
const config = appConfig.getConfig()
```

### 环境变量迁移

原有的 `.env` 文件配置仍然有效，作为默认值使用。建议：

1. 保留 `.env` 文件作为默认配置
2. 在设置页面中覆盖需要修改的配置
3. 敏感信息（如 API Key）优先在设置页面配置

## 最佳实践

1. **开发环境**：使用 `.env` 文件配置
2. **生产环境**：通过设置页面配置，避免暴露敏感信息
3. **团队协作**：使用导出/导入功能共享配置模板
4. **配置备份**：定期导出配置文件备份

## 注意事项

1. **配置存储位置**：
   - WPS 环境：保存在用户数据目录的 JSON 文件中
   - 浏览器环境：降级到 `localStorage`（仅用于开发调试）
2. **安全性**：敏感信息（API Key、Token）以明文存储在配置文件中，请注意文件权限
3. **配置生效**：配置更新后，某些服务可能需要重新初始化才能生效
4. **导入操作**：导入配置会覆盖当前所有配置，请谨慎操作
5. **配置文件**：可以手动编辑 `wps_addon_unified_config.json` 文件

## 故障排查

### 配置未生效
1. 检查是否点击了"保存配置"按钮
2. 刷新页面或重启 WPS
3. 检查浏览器控制台是否有错误信息

### 配置丢失
1. 检查配置文件是否存在：`%UserDataPath%\wps_addon_config.json`
2. 尝试从备份的配置文件导入
3. 重新在设置页面配置

### 查看配置文件位置
在 WPS 中打开控制台，执行：
```javascript
console.log(window.Application.Env.GetUserDataPath())
```

### API 调用失败
1. 检查 API Key 和 URL 是否正确
2. 检查网络连接
3. 查看控制台错误日志
