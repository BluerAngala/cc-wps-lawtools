# 配置动态更新说明

## 问题描述

在实现统一配置管理后，发现了一个关键问题：**用户在设置页面修改 AI 配置后，合同审查功能并不会立即使用新配置**。

### 问题根源

1. **AI 客户端在模块加载时就创建**：`siliconflow.js` 在模块加载时就创建了 `apiClient` 实例
2. **配置更新后客户端未刷新**：用户修改配置保存后，`apiClient` 仍然使用旧的 API Key 和 baseUrl
3. **需要刷新页面才能生效**：只有重新加载整个插件，才会重新创建 `apiClient` 实例

## 解决方案

### 1. AI 配置动态更新

**修改文件：** `src/views/SettingsPage.vue`

在以下场景中调用 `reinitializeAIClient()` 方法：

#### (1) 自动保存配置时
```javascript
// 自动保存配置
const autoSave = () => {
  try {
    appConfig.saveConfig(config.value)
    // 当 AI 配置更新时，重新初始化 AI 客户端
    if (activeTab.value === 'ai') {
      reinitializeAIClient()
      console.log('AI 配置已更新，客户端已重新初始化')
    }
  } catch (error) {
    console.error('保存配置失败:', error)
  }
}
```

#### (2) 重置 AI 标签页时
```javascript
const resetTab = (tab) => {
  // ...
  config.value[tab] = defaults[tab]
  autoSave()
  
  // 如果重置的是 AI 配置，需要重新初始化客户端
  if (tab === 'ai') {
    reinitializeAIClient()
  }
  
  window.$message?.success('已恢复默认配置')
}
```

#### (3) 重置所有配置时
```javascript
const handleResetAll = () => {
  if (confirm('确定要重置所有配置吗？此操作不可恢复。')) {
    config.value = appConfig.reset()
    reinitializeAIClient() // 重置后需要重新初始化 AI 客户端
    window.$message?.success('所有配置已重置')
  }
}
```

#### (4) 导入配置时
```javascript
const handleImport = async (options) => {
  try {
    const importedConfig = await appConfig.importConfig(options.file.file)
    config.value = importedConfig
    reinitializeAIClient() // 导入配置后需要重新初始化 AI 客户端
    window.$message?.success('配置已导入')
  } catch (error) {
    window.$message?.error('导入失败')
  }
  return false
}
```

### 2. 金山文档配置已支持动态更新

**文件：** `src/services/kdocs/kdocs.js`

金山文档服务已经正确实现了动态配置读取：

```javascript
async function kdocsHandler({ type, recordID, sheetID, inputData, isAsync = false }) {
  // 动态获取最新配置
  const config = getKdocsConfig()
  const currentWebhookUrl = config.webhookUrl
  const currentToken = config.token
  
  // ... 使用最新配置
}
```

每次调用 `kdocsHandler` 时都会调用 `getKdocsConfig()` 获取最新配置，**无需额外处理**。

## 技术实现

### AI 客户端重新初始化机制

**文件：** `src/services/ai/siliconflow.js`

```javascript
// 创建 axios 实例的工厂方法
const createApiClient = () => {
  const config = getAIConfig() // 从 appConfig 获取最新配置
  return axios.create({
    baseURL: config.baseUrl,
    timeout: config.timeout,
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${config.apiKey}`
    }
  })
}

let apiClient = createApiClient()

// 提供重新初始化方法
export const reinitializeAIClient = () => {
  apiClient = createApiClient()
  console.log('AI 客户端已重新初始化')
}
```

### 配置获取机制

**文件：** `src/services/ai/siliconflow.js`

```javascript
// 获取 AI 配置（优先使用统一配置，回退到环境变量）
const getAIConfig = () => {
  const config = appConfig.get('ai')
  return {
    apiKey: config.apiKey || import.meta.env.VITE_AI_API_KEY,
    baseUrl: config.baseUrl || import.meta.env.VITE_AI_API_BASE_URL,
    timeout: config.timeout || 30000
  }
}
```

## 使用流程

### 用户操作流程

1. 用户打开设置页面
2. 修改 AI API Key 或其他配置
3. 配置自动保存
4. **AI 客户端自动重新初始化**（无需刷新页面）
5. 回到合同审查页面
6. **立即使用新配置进行 AI 调用**

### 配置生效时机

| 操作 | AI 配置 | 金山文档配置 | 系统配置 |
|------|---------|--------------|----------|
| 实时修改 | ✅ 立即生效 | ✅ 立即生效 | ✅ 立即生效 |
| 重置标签页 | ✅ 立即生效 | ✅ 立即生效 | ✅ 立即生效 |
| 重置所有配置 | ✅ 立即生效 | ✅ 立即生效 | ✅ 立即生效 |
| 导入配置 | ✅ 立即生效 | ✅ 立即生效 | ✅ 立即生效 |

## 测试验证

### 验证步骤

1. **初始状态测试**
   - 打开设置页面，查看当前 AI 配置
   - 打开浏览器控制台
   - 切换到合同审查页面，执行一次 AI 分析
   - 观察控制台输出的 API 调用信息

2. **配置更新测试**
   - 返回设置页面
   - 修改 API Key（输入一个测试值）
   - 观察控制台是否输出 "AI 配置已更新，客户端已重新初始化"
   - 切换到合同审查页面
   - 执行 AI 分析
   - 验证使用的是新的 API Key

3. **配置重置测试**
   - 在设置页面点击"恢复默认"
   - 验证 AI 客户端是否重新初始化
   - 执行 AI 分析，验证配置生效

4. **金山文档测试**
   - 修改金山文档配置
   - 提交合同数据
   - 验证使用的是最新配置

## 注意事项

1. **无需刷新页面**：配置更新后立即生效，用户体验更流畅
2. **向后兼容**：如果配置文件不存在，会自动使用环境变量中的默认值
3. **错误处理**：如果新配置无效（如 API Key 错误），调用时会收到错误提示
4. **日志输出**：所有配置更新都有控制台日志，方便调试
