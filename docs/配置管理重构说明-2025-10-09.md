# 配置管理系统重构说明

## 📅 重构日期
2025-10-09

## 🎯 重构目标
解决配置文件无法正确保存到 WPS 文件系统的问题，确保使用正确的 WPS FileSystem API。

## ❌ 原问题

### 1. API 使用错误
```javascript
// ❌ 错误用法 1
window.Application.FileSystem.WriteFile(fileName, jsonString)  // WriteFile 只接受一个参数
window.Application.FileSystem.ReadFile(fileName)  // ReadFile 功能不明确

// ❌ 错误用法 2（最关键的错误！）
const fullPath = 'C:\\Users\\xxx\\AppData\\Local\\Temp\\wps_addon_config.json'
window.Application.FileSystem.writeFileString(fullPath, jsonString)
// Error: path cannot contains "/" or "\"
```

**问题**：
- `WriteFile(path)` 只用于创建空文件，**不接受第二个参数**
- `ReadFile(path)` 返回值不明确
- **最关键**：`writeFileString` 和 `readFileString` **只接受文件名，不接受包含路径的完整路径**

### 2. 路径处理错误
```javascript
// ❌ 错误：将完整路径传给 API
const tempPath = 'C:\\Users\\xxx\\AppData\\Local\\Temp'
const configPath = tempPath + '\\wps_addon_config.json'
window.Application.FileSystem.writeFileString(configPath, data)
// Error: path cannot contains "/" or "\"
```

**问题**：
- WPS FileSystem API 只接受文件名，文件会自动保存到临时目录
- 传入包含路径的字符串会导致错误
- 没有区分"API 调用参数"和"显示路径"的不同用途

### 3. 缺少错误处理和调试
- 保存后没有验证文件是否真的存在
- 缺少详细的调试信息输出

## ✅ 重构方案

### 1. 使用正确的 API 和参数格式

```javascript
// ✅ 正确用法：只传文件名
const fileName = 'wps_addon_config.json'
window.Application.FileSystem.writeFileString(fileName, jsonString)  // 写入
window.Application.FileSystem.readFileString(fileName)  // 读取
```

**改进**：
- `writeFileString(fileName, data)` - **只传文件名**，写入字符串数据
- `readFileString(fileName)` - **只传文件名**，返回文件内容字符串
- 文件会自动保存到临时目录（通过 `tmpdir()` 获取）

**关键发现**：
> WPS FileSystem API 的路径参数不能包含 `/` 或 `\`，只接受纯文件名！  
> 错误信息：`Error: path cannot contains "/" or "\"`

### 2. 正确的路径处理（区分 API 参数和显示路径）

```javascript
// ✅ 获取配置文件路径（用于 API 调用）
// 注意：只返回文件名，不包含路径！
getConfigPath() {
  if (!this.isWPSAvailable()) {
    return null  // 降级到 localStorage
  }
  // 只返回文件名，WPS 会自动保存到临时目录
  return this.configFileName  // 'wps_addon_config.json'
}

// ✅ 获取完整路径（仅用于显示给用户）
getConfigFullPath() {
  const tempPath = this.getTempPath()
  if (!tempPath) return null
  
  const normalizedPath = tempPath.replace(/[/\\]+$/, '')
  return normalizedPath + '\\' + this.configFileName
}

// ✅ 多重回退策略获取临时目录（仅用于显示）
getTempPath() {
  // 策略 1: FileSystem.tmpdir() (推荐)
  if (window.Application.FileSystem.tmpdir) {
    return window.Application.FileSystem.tmpdir()
  }
  
  // 策略 2: Env.GetTempPath()
  if (window.Application.Env.GetTempPath) {
    return window.Application.Env.GetTempPath()
  }
  
  // 策略 3: Env.GetUserDataPath()
  if (window.Application.Env.GetUserDataPath) {
    return window.Application.Env.GetUserDataPath()
  }
  
  return null  // 降级到 localStorage
}
```

**关键点**：
- `getConfigPath()` - 返回文件名，用于 API 调用
- `getConfigFullPath()` - 返回完整路径，仅用于显示给用户
- `getTempPath()` - 获取临时目录，仅用于拼接显示路径

### 3. 增强的错误处理

```javascript
// ✅ 完整的错误处理和验证
  saveConfig(config) {
    try {
      const configPath = this.getConfigPath()  // 只返回文件名
      
      if (!configPath) {
        localStorage.setItem('wps_addon_config', JSON.stringify(config))
        console.log('配置已保存到 localStorage')
        return true
      }

      // 使用 writeFileString 写入，只传文件名
      const jsonString = JSON.stringify(config, null, 2)
      window.Application.FileSystem.writeFileString(configPath, jsonString)
      
      // 显示完整路径给用户
      const fullPath = this.getConfigFullPath()
      console.log('配置已保存到:', fullPath || configPath)
      
      // 验证保存（仍然只用文件名）
      const exists = window.Application.FileSystem.Exists(configPath)
      if (!exists) {
        console.error('保存后文件不存在！')
        return false
      }
      
      return true
    } catch (error) {
      console.error('保存配置失败:', error)
      console.error('错误详情:', error.message)
      return false
    }
  }
```

### 4. 新增调试功能

```javascript
// ✅ 内置调试方法

  console.log('3. 配置文件路径:', this.getConfigPath())
  
  const configPath = this.getConfigPath()

    console.log('4. 文件存在:', window.Application.FileSystem.Exists(configPath))

    }
  } else {
    console.log('4. 使用 localStorage')
    console.log('5. localStorage 内容:', localStorage.getItem('wps_addon_config'))
  }
  
  console.log('6. 当前配置:', this.getConfig())
}
```

## 🔧 WPS FileSystem API 对照表

| 功能 | ❌ 错误用法 | ✅ 正确用法 | 参数格式 |
|------|-----------|-----------|---------|
| 写入字符串 | `WriteFile(path, data)` | `writeFileString(fileName, data)` | **只传文件名** |
| 读取字符串 | `ReadFile(path)` | `readFileString(fileName)` | **只传文件名** |
| 创建空文件 | - | `WriteFile(fileName)` | **只传文件名** |
| 检查文件存在 | `Exists(fullPath)` ❌ | `Exists(fileName)` | **只传文件名** |
| 删除文件 | `Remove(fullPath)` ❌ | `unlinkSync(fileName)` | **只传文件名** |
| 获取临时目录 | - | `FileSystem.tmpdir()` | 返回完整路径（仅用于显示） |

### ⚠️ 关键规则

> **所有 FileSystem 方法的路径参数都不能包含 `/` 或 `\`，只接受纯文件名！**
> 
> 文件会自动保存到临时目录，通过 `tmpdir()` 可以获取该目录的完整路径（仅用于显示）。
