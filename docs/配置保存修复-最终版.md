# 配置保存修复 - 最终版本

## 🎉 问题已解决！

### 核心问题

WPS FileSystem API 的路径参数**不能包含路径分隔符 `/` 或 `\`**，只接受纯文件名！

```
Error: path cannot contains "/" or "\"
```

### 错误代码

```javascript
// ❌ 错误：传入完整路径
const fullPath = 'C:\\Users\\xxx\\AppData\\Local\\Temp\\wps_addon_config.json'
window.Application.FileSystem.writeFileString(fullPath, jsonString)
// Error: path cannot contains "/" or "\"
```

### 正确代码

```javascript
// ✅ 正确：只传文件名
const fileName = 'wps_addon_config.json'
window.Application.FileSystem.writeFileString(fileName, jsonString)
// 文件会自动保存到临时目录
```

## 🔑 关键改动

### 1. 区分两种路径

```javascript
// 用于 API 调用 - 只返回文件名
getConfigPath() {
  return this.configFileName  // 'wps_addon_config.json'
}

// 用于显示给用户 - 返回完整路径
getConfigFullPath() {
  const tempPath = this.getTempPath()
  return tempPath + '\\' + this.configFileName
  // 'C:\\Users\\xxx\\AppData\\Local\\Temp\\wps_addon_config.json'
}
```

### 2. API 调用规则

| API 方法 | 参数 | 说明 |
|---------|------|------|
| `writeFileString(fileName, data)` | **只传文件名** | 写入字符串到文件 |
| `readFileString(fileName)` | **只传文件名** | 读取文件内容 |
| `Exists(fileName)` | **只传文件名** | 检查文件是否存在 |
| `unlinkSync(fileName)` | **只传文件名** | 删除文件 |
| `tmpdir()` | 无参数 | 获取临时目录路径（用于显示） |

### 3. 正确的保存流程

```javascript
saveConfig(config) {
  // 1. 获取文件名（用于 API）
  const fileName = this.getConfigPath()  // 'wps_addon_config.json'
  
  // 2. 写入文件（只传文件名）
  const jsonString = JSON.stringify(config, null, 2)
  window.Application.FileSystem.writeFileString(fileName, jsonString)
  
  // 3. 获取完整路径（用于日志显示）
  const fullPath = this.getConfigFullPath()
  console.log('配置已保存到:', fullPath)
  // 输出: 配置已保存到: C:\Users\xxx\AppData\Local\Temp\wps_addon_config.json
  
  // 4. 验证保存（只传文件名）
  const exists = window.Application.FileSystem.Exists(fileName)
  return exists
}
```

## 🎓 经验教训

### 重要发现

> **WPS FileSystem API 的设计哲学**：
> 
> 文件操作只需要文件名，系统会自动管理文件存储位置（临时目录）。
> 这样设计可以避免路径相关的安全和兼容性问题。

### API 使用原则

1. **文件操作**：只传文件名
2. **路径获取**：用于显示和日志
3. **不要混用**：API 参数 ≠ 显示路径

### 调试技巧

当遇到 WPS API 错误时：
1. 仔细阅读错误信息
2. 查看 WPS 官方文档
3. 使用 `console.log` 验证参数
4. 测试最简单的用例
