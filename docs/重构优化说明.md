# 代码重构优化说明

## 重构目标
解决代码冗余重复问题，提高代码可维护性和可读性。

## 主要改进

### 1. 创建统一服务管理器
- **新增文件**: `src/utils/contractService.js`
- **功能**: 整合所有合同相关功能的统一入口
- **优势**: 减少重复代码，提供一致的API接口

### 2. 简化主组件 ContractServices.vue
**重构前**:
- 292行代码
- 包含大量重复的任务处理逻辑
- 配置管理代码冗余
- 任务结果处理器映射复杂

**重构后**:
- 126行代码（减少57%）
- 统一使用 contractService
- 简化的事件处理方法
- 清晰的职责分离

### 3. 优化工具类

#### ConfigManager (src/utils/configManager.js)
- 统一配置管理逻辑
- 支持默认配置和配置合并
- 简化保存/加载流程

#### TaskManager (src/utils/taskManager.js)
- 提取文档内容获取为独立方法
- 使用可选链操作符简化代码
- 减少重复的错误处理逻辑

#### DataSubmitter (src/utils/dataSubmitter.js)
- 将数据处理拆分为多个小方法
- 复用字段标准化逻辑
- 减少重复的数据验证代码

### 6. 使用示例

#### 重构前 - 执行任务
```javascript
// 复杂的任务执行逻辑
const executeTask = async (ruleType, params = {}) => {
  if (processingRules.value.has(ruleType)) {
    ElMessage.warning('该任务正在执行中，请稍候...')
    return
  }
  processingRules.value.add(ruleType)
  try {
    await taskManager.executeTask(
      ruleType, params,
      (result, taskParams) => { handleTaskResult(ruleType, result, taskParams) },
      (error) => { ElMessage.error(error.message || '任务执行失败') }
    )
  } catch (error) {
    ElMessage.error(error.message || '任务执行失败，请检查文档状态。')
  } finally {
    processingRules.value.delete(ruleType)
  }
}
```

#### 重构后 - 执行任务
```javascript
// 简化的任务执行
const executeExtraction = (config) => {
  contractService.executeTask('extractText', config, (result) => {
    const processedData = contractService.processExtractedData(result)
    if (processedData) {
      extractedData.value = processedData
    }
  })
}
```
