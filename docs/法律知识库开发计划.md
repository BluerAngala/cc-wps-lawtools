## Phase 1：知识管理界面

**目标**：在 ChatSettings 的「向量检索」Tab 中提供法律知识的基础管理能力。

### 功能点

- [ ] 知识条目列表（分页、搜索、按类别筛选）
- [ ] 手动添加单条知识（标题、内容、来源、类别）
- [ ] 删除单条 / 批量删除
- [ ] 批量导入（粘贴 JSON 或上传文件）
- [ ] 统计面板（总量、各类别数量、最近更新时间）

### 涉及文件

| 文件 | 变更 |
|------|------|
| `qdrantClient.js` | 新增 `scrollByFilter(collection, filter, limit, offset)` 支持分页浏览 |
| `ragService.js` | 新增 `listLawKnowledge(offset, limit, category)` 分页查询 |
| `ChatSettings.vue` | 「向量检索」Tab 增加「法律知识库」子区域 |

### 数据结构

```javascript
{
  id: number,           // _generatePointId(title + content)
  vector: float[],      // 1024 维
  payload: {
    title: '合同法第九十四条',

    source: '中华人民共和国合同法',

  }
```

## Phase 2：结构化数据源导入

**目标**：从权威法律数据库批量导入，而非手工录入。

### 数据源优先级

| 优先级 | 数据源 | 内容 | 导入方式 |
|--------|--------|------|---------|
| P0 | 国家法律法规数据库 (flk.npc.gov.cn) | 法律、行政法规、司法解释 | 爬取 → 结构化 → 分块入库 |
| P1 | 常用合同审查法条精选 | 合同法高相关条款（~200条） | 手工 JSON + 自动入库 |
| P2 | 地方法院裁判指引 | 审判实务指导 | 爬取 → 结构化 |
| P3 | 企业内部合规制度 | 公司特有的合规规范 | 文档上传 → 分块入库 |

### 导入流程设计

```
数据源 → 结构化提取 → 按条/按款分割
    │
    ▼
每条生成 { title, content, source, category }
    │
    ▼
Embedding 批量嵌入（每批 20 条）
    │
    ▼
Upsert 到 law_knowledge 集合
    │
    ▼
记录导入日志（来源、条数、时间）
```

### 涉及文件

| 文件 | 变更 |
|------|------|
| `scripts/import-law-knowledge.mjs` | **新建** — Node.js 离线导入脚本 |
| `ragService.js` | 新增 `batchIndexLawKnowledge(entries)` 优化大批量写入 |
| `embeddingService.js` | 批量接口优化（并发控制 + 错误重试） |

### 离线脚本用法（预想）

```bash
# 从 JSON 文件导入
node scripts/import-law-knowledge.mjs --file ./data/contract-law.json

# 从目录批量导入
node scripts/import-law-knowledge.mjs --dir ./data/laws/
```

JSON 文件格式：
```json
[
  {
    "title": "合同法第九十四条",
    "content": "有下列情形之一的，当事人可以解除合同：...",
    "source": "中华人民共和国合同法",
    "category": "解除权"
  }
]
```

## Phase 3：智能检索增强

**目标**：优化检索质量，让 RAG 上下文更精准。

### 功能点

- [ ] 混合检索（向量 + BM25 关键词），提升精确术语匹配
- [ ] 检索结果重排序（Reranker），Top-N 精排
- [ ] 查询改写（Query Rewriting），将口语化问题转为法律术语
- [ ] 知识去重与版本控制（同一法条新旧版本）
- [ ] 检索结果缓存（同一会话内相同 query 复用）

### 涉及文件

| 文件 | 变更 |
|------|------|
| `ragService.js` | 新增 `hybridSearch()` 混合检索方法 |
| `chatService.js` | 检索环节改用混合检索 + Reranker |
| `promptTemplates.js` | RAG 上下文区优化格式（区分法律依据 vs 文档段落） |
