# 配置文件存储说明

## 📁 配置文件位置

配置文件采用**双重存储策略**，确保在不同环境下都能正常工作。

### 1. **主存储方式：WPS 用户数据目录**（推荐）

当 WPS 环境可用时，配置文件保存在 WPS 的用户数据目录：

```
文件路径：{WPS用户数据目录}\wps_addon_config.json
```

#### 获取方式
```javascript
const userPath = window.Application.Env.GetUserDataPath()
const configPath = userPath + '\\wps_addon_config.json'
```

#### 典型路径示例
```
C:\Users\{用户名}\AppData\Roaming\Kingsoft\office6\wps_addon_config.json
```

或者

```
C:\Users\{用户名}\AppData\Local\Kingsoft\WPS Office\wps_addon_config.json
```

> **注意**：具体路径取决于 WPS 的安装方式和版本，但都在用户的 AppData 目录下。

---

### 2. **备用存储方式：浏览器 localStorage**

当 WPS API 不可用时（如在浏览器中调试），配置会降级保存到浏览器的 localStorage：

```
存储键：wps_addon_config
位置：浏览器的 localStorage
```

#### 查看方式
在浏览器开发者工具中：
1. 按 F12 打开开发者工具
2. 切换到 "Application" 或 "应用" 标签
3. 左侧找到 "Local Storage"
4. 查找 `wps_addon_config` 键

## 🔐 配置文件安全性

### 安全特性

1. **本地存储**：配置文件保存在用户本地，不会上传到服务器
2. **用户隔离**：每个 Windows 用户有独立的配置文件
3. **敏感信息**：API Key 等敏感信息仅存储在本地

### 安全建议

1. ⚠️ **不要分享配置文件**：包含 API Key 等敏感信息
2. ⚠️ **定期更换密钥**：如果怀疑 API Key 泄露，及时更换
3. ✅ **使用导出功能**：导出配置前，建议先删除敏感信息

## 🛠️ 配置文件操作

### 查看配置文件

**Windows 文件资源管理器：**
```
1. Win + R 打开运行
2. 输入：%APPDATA%\Kingsoft
3. 找到 wps_addon_config.json
4. 用记事本打开查看
```

### 手动编辑配置文件

1. **关闭 WPS**（确保配置不会被覆盖）
2. 用文本编辑器打开配置文件
3. 修改后保存
4. 重新打开 WPS

> ⚠️ **注意**：手动编辑时必须保持 JSON 格式正确，否则配置可能无法加载。

### 备份配置文件

**方法 1：通过设置页面**
- 打开设置页面
- 点击"导出配置"按钮

**方法 1：通过设置页面**

- 点击"导入配置"按钮
- 选择之前导出的 JSON 文件

**方法 2：手动复制**
- 将备份的 `wps_addon_config.json` 复制回原位置

- 找到配置文件位置

- 重启插件会自动创建新的默认配置

## 🔄 配置文件工作流程

### 保存流程

```
用户修改配置
    ↓
触发 autoSave()
    ↓
调用 appConfig.saveConfig()
    ↓
获取配置路径（WPS 用户数据目录）
    ↓
使用 FileSystemObject 写入文件
    ↓
JSON.stringify() 格式化
    ↓
保存为 wps_addon_config.json
```

### 读取流程

```
插件启动 / 页面加载
    ↓
调用 appConfig.getConfig()
    ↓
获取配置路径
    ↓
检查文件是否存在
    ↓
读取文件内容
    ↓
JSON.parse() 解析
    ↓
与默认配置合并（确保新字段存在）
    ↓
返回完整配置对象
```

### 降级流程

```
尝试访问 WPS API
    ↓
API 不可用
    ↓
降级到 localStorage
    ↓
使用 localStorage.getItem('wps_addon_config')
    ↓
返回配置对象
```

## ❓ 常见问题

### Q1: 找不到配置文件？
**A:** 可能是因为：
1. 还没有保存过配置（首次使用）
2. 配置保存在 localStorage（浏览器调试模式）
3. WPS 安装路径不同

### Q2: 配置没有生效？
**A:** 请检查：
1. 配置文件 JSON 格式是否正确
2. 是否已保存（输入框失去焦点时自动保存）
3. AI 配置是否已重新初始化（查看控制台日志）

### Q3: 如何迁移配置到其他电脑？
**A:** 
1. 在原电脑上导出配置
2. 将 JSON 文件复制到新电脑
3. 在新电脑上导入配置

### Q4: 配置文件会被同步吗？
**A:** 
- 不会自动同步
- 配置仅保存在本地
- 需要手动导出/导入来迁移

## 🔧 技术实现

### 相关代码

```javascript
// src/utils/appConfig.js

class AppConfigManager {
  constructor() {
    this.configFile = 'wps_addon_config.json'  // 配置文件名
  }

  getConfigPath() {
    try {
      if (typeof window.Application === 'undefined') {
        return null  // 降级到 localStorage
      }
      const userPath = window.Application.Env.GetUserDataPath()
      return userPath + '\\' + this.configFile
    } catch (error) {
      console.error('获取配置路径失败:', error)
      return null
    }
  }

  saveConfig(config) {
    try {
      const configPath = this.getConfigPath()
      
      // 降级处理
      if (!configPath) {
        localStorage.setItem('wps_addon_config', JSON.stringify(config))
        return true
      }

      // 使用 WPS API 保存
      const fso = new ActiveXObject('Scripting.FileSystemObject')
      const file = fso.CreateTextFile(configPath, true)
      file.Write(JSON.stringify(config, null, 2))
      file.Close()
      
      return true
    } catch (error) {
      console.error('保存配置失败:', error)
      return false
    }
  }
}
```
