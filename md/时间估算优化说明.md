# 时间估算优化说明

## 优化背景

原有的时间估算存在以下问题：
1. **显示固定时间**：只显示一个确定的数字（如 "30 秒"）
2. **容易误导用户**：实际时间经常超出预估
3. **没有预留空间**：未考虑网络波动、AI 响应慢等情况
4. **用户体验差**：超时后用户会感到被欺骗

## 优化方案

### 改为时间区间显示

#### 原方案
```javascript
const estimatedTime = Math.ceil(wordCount / 500) * 10  // 固定值
window.$message?.info(`预计用时: ${estimatedTime} 秒`)
```
**问题**：给出确定时间，但实际往往会超时

#### 新方案
```javascript
const minTime = Math.ceil(wordCount / 600) * 10  // 最快情况
const maxTime = Math.ceil(wordCount / 400) * 15  // 预留充足时间
const timeRange = `${minTime}-${maxTime} 秒`
window.$message?.info(`预计用时: ${timeRange}`)
```
**改进**：给出时间区间，更符合实际情况

### 时间估算算法

#### 全文模式
```javascript
最快时间 = Math.ceil(字数 / 600) * 10 秒
最慢时间 = Math.ceil(字数 / 400) * 15 秒
```

- **每 600 字最快 10 秒**：理想网络环境下的速度
- **每 400 字最慢 15 秒**：预留充足的缓冲时间

#### 分段模式
```javascript
最快时间 = Math.ceil(字数 / 1200) * 8 秒
最慢时间 = Math.ceil(字数 / 800) * 12 秒
```

- **每 1200 字最快 8 秒**：分段处理更快
- **每 800 字最慢 12 秒**：预留缓冲时间

### 自动单位转换

当时间超过 60 秒时，自动转换为分钟：

```javascript
if (maxTime <= 60) {
  timeRange = `${minTime}-${maxTime} 秒`
} else {
  const minMinutes = Math.ceil(minTime / 60)
  const maxMinutes = Math.ceil(maxTime / 60)
  timeRange = `${minMinutes}-${maxMinutes} 分钟`
}
```

## 技术细节

### 1. 时间计算逻辑

```javascript
// 计算预计时间区间
const wordCount = fullText.length
let timeRange = ''

if (scanStrategy.value === 'full') {
  // 全文模式：根据字数估算时间区间
  const minTime = Math.ceil(wordCount / 600) * 10  // 最快情况
  const maxTime = Math.ceil(wordCount / 400) * 15  // 预留充足时间
  
  if (maxTime <= 60) {
    timeRange = `${minTime}-${maxTime} 秒`
  } else {
    const minMinutes = Math.ceil(minTime / 60)
    const maxMinutes = Math.ceil(maxTime / 60)
    timeRange = `${minMinutes}-${maxMinutes} 分钟`
  }
} else {
  // 分段模式：通常更快
  const minTime = Math.ceil(wordCount / 1200) * 8
  const maxTime = Math.ceil(wordCount / 800) * 12
  
  if (maxTime <= 60) {
    timeRange = `${minTime}-${maxTime} 秒`
  } else {
    const minMinutes = Math.ceil(minTime / 60)
    const maxMinutes = Math.ceil(maxTime / 60)
    timeRange = `${minMinutes}-${maxMinutes} 分钟`
  }
}
```

## 最佳实践

### 1. 宁可多不可少
- 预留足够的时间缓冲
- 宁愿让用户惊喜（提前完成）
- 不要让用户失望（超时）

### 2. 区间要合理
- 不要太宽（如 1-60 分钟），失去指导意义
- 不要太窄（如 10-12 秒），容易超出
- 推荐比例：最大时间 ≈ 最小时间 × 1.5-2.5

### 3. 根据模式区分
- 全文模式：更全面但更慢，预留更多时间
- 分段模式：更快但分步骤，时间更可控

### 4. 动态调整
- 根据用户反馈调整系数
- 收集实际耗时数据
- 持续优化算法
