# 敏感信息脱敏功能修复说明

## 问题描述

敏感信息脱敏功能总是识别未发现敏感信息。

## 问题原因

1. **正则表达式过于宽泛**：手机号正则 `/1[3-9]\d{9}/g` 会匹配身份证号中以1开头的11位数字
2. **重复匹配问题**：身份证号会被银行卡号规则重复匹配
3. **缺少边界检测**：没有使用负向前瞻和负向后瞻来避免误匹配

## 修复方案

### 1. 优化正则表达式

使用 **负向前瞻 `(?!\d)` 和负向后瞻 `(?<!\d)`** 来确保数字前后没有其他数字：

```javascript
// 修复前
regex: /1[3-9]\d{9}/g

// 修复后
regex: /(?<!\d)1[3-9]\d{9}(?!\d)/g
```

### 2. 避免重复匹配

使用 `Set` 记录已匹配的文本，避免同一文本被多个规则匹配：

```javascript
const matchedTexts = new Set()

SENSITIVE_PATTERNS.forEach((pattern) => {
  matches.forEach((match) => {
    if (matchedTexts.has(match)) {
      return // 跳过已匹配的文本
    }
    // ... 处理逻辑
    matchedTexts.add(match)
  })
})
```

### 3. 优化银行卡号规则

在银行卡号的 `mask` 函数中排除身份证号：

```javascript
mask: (match) => {
  // 排除身份证号（18位且符合身份证格式）
  if (match.length === 18 && /^[1-9]\d{5}(18|19|20)\d{2}.../.test(match)) {
    return match // 不处理，留给身份证号规则
  }
  return match.replace(/(\d{4})\d+(\d{4})/, '$1****$2')
}
```

## 修复后的效果

### 测试用例

```javascript
const text = '我的手机号是13800138000，身份证号是110101199001011234，邮箱是test@example.com，姓名：张三'
```

### 检测结果

- ✅ 身份证号: `110101199001011234` → `110101********1234`
- ✅ 手机号: `13800138000` → `138****8000`
- ✅ 邮箱: `test@example.com` → `t***@example.com`
- ✅ 姓名: `姓名：张三` → `姓名：张*`

### 边界情况测试

- ✅ 空文档：正确返回 0 个
- ✅ 无敏感信息：正确返回 0 个
- ✅ 多个相同手机号：正确去重
- ✅ 连续数字（如编号）：不会误匹配
- ✅ 真实场景（多人信息）：正确识别所有敏感信息

## 技术要点

1. **负向前瞻/后瞻**：`(?<!\d)` 和 `(?!\d)` 确保数字边界
2. **去重机制**：使用 `Set` 避免重复匹配
3. **优先级处理**：身份证号优先于银行卡号匹配
4. **正则转义**：使用 `replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` 转义特殊字符

## 额外修复：应用脱敏时保留格式

### 问题
直接设置 `doc.Range().Text` 会丢失所有格式（字体、颜色、段落等）。

### 解决方案
使用 WPS 的 **Find & Replace API** 进行替换，保留原有格式：

```javascript
const selection = doc.Application.Selection
const findObj = selection.Find

findObj.Text = item.original
findObj.Replacement.Text = item.desensitized
findObj.Execute(undefined, undefined, undefined, undefined, undefined, 
                undefined, undefined, undefined, undefined, undefined, 
                2) // wdReplaceAll = 2，替换全部
```

### 优势
- ✅ 保留字体格式
- ✅ 保留颜色样式
- ✅ 保留段落格式
- ✅ 保留表格结构
- ✅ 不影响其他内容
