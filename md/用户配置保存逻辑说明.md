# WPS 插件用户配置保存逻辑说明

## 📋 概述

本项目使用统一的配置管理系统来保存和读取用户设置，核心是 `AppConfigManager` 类，通过 WPS FileSystem API 或浏览器 localStorage 进行持久化存储。

---

## 🏗️ 核心架构

### 1. 配置管理器 (`src/utils/appConfig.js`)

**单例实例**：
```javascript
export const appConfig = new AppConfigManager()
```

### 2. 存储策略

#### WPS FileSystem API（唯一存储方式）
- **存储位置**：WPS 临时目录
- **文件名**：`wps_addon_config.json`
- **获取路径策略**（多重回退）：
  1. `window.Application.FileSystem.tmpdir()` - 推荐
  2. `window.Application.Env.GetTempPath()` - 备选方案 1
  3. `window.Application.Env.GetUserDataPath()` - 备选方案 2
- **注意**：必须在 WPS 环境中运行，非 WPS 环境将无法保存配置

## 🔄 核心方法说明

### 1. 读取配置 - `getConfig()`

**流程**：
```
1. 检查 WPS 环境是否可用
   ├─ 是 → 尝试从 WPS 文件系统读取
   │   ├─ 文件存在 → 读取并与默认配置深度合并
   │   └─ 文件不存在 → 返回默认配置
   └─ 否 → 返回默认配置（警告：WPS 环境不可用）
```

**代码示例**：
```javascript
// 读取完整配置
const config = appConfig.getConfig()

// 读取指定分类配置
const aiConfig = appConfig.get('ai')
const kdocsConfig = appConfig.get('kdocs')
```

### 2. 保存配置 - `saveConfig(config)`

**流程**：
```
1. 检查 WPS 环境是否可用
   ├─ 是 → 使用 writeFileString() 保存到 JSON 文件
   │   └─ 验证文件是否成功创建
   └─ 否 → 保存失败（提示：WPS 环境不可用）
```

**代码示例**：
```javascript
// 保存完整配置
const success = appConfig.saveConfig(config)

// 保存指定分类配置
appConfig.set('ai', { apiKey: 'xxx', ... })
```

### 3. 重置配置 - `reset()`

**操作**：
- 删除 WPS 文件系统中的配置文件
- 返回默认配置对象

### 4. 深度合并 - `deepMerge(target, source)`

**作用**：
- 将用户保存的配置与默认配置合并
- 确保新增的配置项也能正常使用
- 处理嵌套对象的合并

## 🎯 使用场景

### 场景 1：设置页面（`src/views/SettingsPage.vue`）

**自动保存机制**：
```javascript
const autoSave = () => {
  try {
    // 保存配置
    appConfig.saveConfig(config.value)
    
    // 如果是 AI 配置，重新初始化客户端
    if (activeTab.value === 'ai') {
      reinitializeAIClient()
    }
  } catch (error) {
    console.error('保存配置失败:', error)
  }
}

// 绑定到输入框的 @update:value 事件
<n-input v-model:value="config.ai.apiKey" @update:value="autoSave" />
```

**特点**：
- ✅ 实时保存：用户修改后立即保存
- ✅ 无需手动点击保存按钮
- ✅ AI 配置更新后自动重新初始化客户端

### 场景 2：AI 服务（`src/services/ai/siliconflow.js`）

**动态读取配置**：
```javascript
const getAIConfig = () => {
  const config = appConfig.get('ai')
  return {
    apiKey: config.apiKey || import.meta.env.VITE_AI_API_KEY,
    baseUrl: config.baseUrl || import.meta.env.VITE_AI_API_BASE_URL,
    timeout: config.timeout || 30000
  }
}

// 创建 axios 客户端
const createApiClient = () => {
  const config = getAIConfig()
  return axios.create({
    baseURL: config.baseUrl,
    timeout: config.timeout,
    headers: {
      Authorization: `Bearer ${config.apiKey}`
    }
  })
}
```

**重新初始化机制**：
```javascript
export const reinitializeAIClient = () => {
  apiClient = createApiClient()
  console.log('AI 客户端已重新初始化')
}
```

### 场景 3：合同服务（`src/services/contract/contractService.js`）

**读取业务配置**：
```javascript
import { appConfig } from '../../utils/appConfig.js'

// 获取提取标签

// 获取关键词列表
```

## ⚠️ 注意事项

### 1. WPS FileSystem API 要求

- ⚠️ **必须在 WPS 环境中运行**：非 WPS 环境将无法保存配置
- ⚠️ **只接受文件名**：不能包含路径分隔符
- ⚠️ **自动保存位置**：文件自动保存到 WPS 临时目录

### 2. 数据安全

- 🔒 敏感信息（API Key、Token）存储在本地
- 🔒 不会上传到服务器
- 🔒 建议使用环境变量作为默认值

### 3. 配置更新

- ✅ AI 配置更新后必须调用 `reinitializeAIClient()`
- ✅ 使用深度合并确保不丢失默认配置
- ✅ 实时保存无需手动触发

### 4. 错误处理

```javascript
try {
  const config = appConfig.getConfig()
  // 使用配置
} catch (error) {
  console.error('读取配置失败:', error)
  // 使用默认配置
}
```

## 📖 总结

本项目的配置保存逻辑具有以下特点：

✅ **WPS 文件系统**：使用 WPS FileSystem API 持久化存储，专为 WPS 加载项优化  
✅ **自动保存**：实时保存用户修改，无需手动点击保存按钮  
✅ **深度合并**：新增配置项不会导致旧数据丢失  
✅ **类型丰富**：支持文本、数字、标签、列表等多种配置类型  
✅ **调试友好**：提供完善的调试工具和日志输出  
✅ **安全可靠**：本地存储，不上传到服务器  

⚠️ **重要提醒**：必须在 WPS 环境中运行，非 WPS 环境将无法保存配置。

通过 `appConfig` 单例实例，整个项目中的所有模块都可以方便地读取和保存用户配置。
