# 模板加载问题修复

**日期：** 2025-11-14  
**类型：** Bug 修复

---

## 🐛 问题描述

用户报告：
1. ❌ 新建分类后，重启 WPS 分类丢失
2. ❌ 保存的自定义模板，重启后消失
3. ❌ 只能看到内置模板，看不到用户模板

---

## 🔍 问题根源

### 原因 1：只加载内置模板

之前的 `loadTemplates()` 方法**只加载内置模板**，没有加载用户保存的模板配置文件：

```javascript
// ❌ 之前的代码 - 只加载内置模板
async loadTemplates() {
  const configUrl = '/templates/templates.json'  // 只读取内置配置
  const response = await fetch(configUrl)
  const data = await response.json()
  return this.processTemplates(data.templates || [])  // 只返回内置模板
}
```

### 原因 2：保存时包含内置模板

之前的 `saveTemplates()` 会保存所有模板（包括内置模板），导致配置文件过大且不必要。

---

## ✅ 解决方案

### 修改 1：合并加载内置模板和用户模板

```javascript
// ✅ 修复后的代码
async loadTemplates() {
  // 1. 加载内置模板
  const builtInTemplates = await this.loadBuiltInTemplates()
  
  // 2. 加载用户保存的模板
  const userTemplates = this.loadUserTemplates()
  
  // 3. 合并模板（用户模板在前）
  return [...userTemplates, ...builtInTemplates]
}
```

### 修改 2：新增 loadBuiltInTemplates() 方法

```javascript
async loadBuiltInTemplates() {
  const configUrl = pathManager.getBuiltInTemplatesPath() + '/templates.json'
  const response = await fetch(configUrl)
  const data = await response.json()
  
  // 处理内置模板路径，标记为内置
  return data.templates.map((template) => ({
    ...template,
    filePath: pathManager.getBuiltInTemplatesPath() + '/' + template.fileName,
    isBuiltIn: true  // 标记为内置模板
  }))
}
```

### 修改 3：新增 loadUserTemplates() 方法

```javascript
loadUserTemplates() {
  if (!pathManager.isWPSAvailable()) return []
  
  const fs = window.Application.FileSystem
  const configPath = pathManager.getTemplatesConfigPath()
  
  // 检查配置文件是否存在
  if (!fs.Exists(configPath)) {
    return []  // 文件不存在，返回空数组
  }
  
  // 读取并解析配置文件
  const content = fs.ReadFile(configPath)
  const data = JSON.parse(content)
  return data.templates || []
}
```

### 修改 4：保存时只保存用户模板

```javascript
saveTemplates(templates) {
  // 过滤掉内置模板，只保存用户模板
  const userTemplates = templates.filter(t => !t.isBuiltIn)
  
  const configData = {
    templates: userTemplates,  // 只保存用户模板
    lastUpdated: new Date().toISOString()
  }
  
  fs.WriteFile(configPath, JSON.stringify(configData, null, 2))
}
```

## 📊 数据流程图

### 保存模板流程

```
用户点击"保存为模板"
    ↓
保存文档到: {ConfigDir}/templates/模板名.docx
    ↓
创建模板配置对象: { name, category, filePath, isBuiltIn: false }
    ↓
添加到 templates 数组
    ↓
调用 saveTemplates(templates)
    ↓
过滤出用户模板: templates.filter(t => !t.isBuiltIn)
    ↓
保存到: {ConfigDir}/templates/templates.json
    ↓
✅ 完成
```

### 加载模板流程

```
页面加载 (onMounted)
    ↓
调用 loadTemplates()
    ↓
并行加载两个来源:
    ├─ loadBuiltInTemplates() → 从 /templates/templates.json 加载（HTTP）
    └─ loadUserTemplates() → 从 {ConfigDir}/templates/templates.json 加载（FileSystem）
    ↓
合并两个数组: [...userTemplates, ...builtInTemplates]
    ↓
✅ 显示所有模板
```

## 🔧 文件路径对比

### 内置模板

| 项目 | 路径 |
|------|------|
| 配置文件 | `/templates/templates.json` (HTTP) |
| 模板文件 | `/templates/xxx.docx` (HTTP) |
| 标记 | `isBuiltIn: true` |
| 特点 | 只读，随项目打包 |

### 用户模板

| 项目 | 路径 |
|------|------|
| 配置文件 | `{AppDataPath}/wps_lawtools_1.0.0_config/templates/templates.json` |
| 模板文件 | `{AppDataPath}/wps_lawtools_1.0.0_config/templates/xxx.docx` |
| 标记 | `isBuiltIn: false` |
| 特点 | 可读写，用户数据 |

## 🎓 关键改进

### 1. 分离加载逻辑

- **内置模板**：从项目资源加载（HTTP）
- **用户模板**：从配置目录加载（FileSystem）
- **合并显示**：用户模板优先

### 2. 分离保存逻辑

- **保存时过滤**：只保存用户模板
- **避免冲突**：不保存内置模板到用户配置

### 3. 明确标记

- **isBuiltIn 标记**：区分内置和用户模板
- **便于过滤**：保存和加载时可以准确区分
