# 内置模板路径修复说明

## 问题描述

打包后内置模板文件无法正常读取，出现 404 错误。

## 根本原因

根据 WPS 插件打包机制：

1. **调试时**：开发服务器本地路径（如 `http://127.0.0.1:3889/`）
2. **打包后**：插件被解压到 `%ProgramData%\Kingsoft\wps\jsaddons\插件名\`，工作目录变了
3. **问题**：使用相对路径（如 `./templates/xxx.json`）在打包后会找不到文件

## 解决方案

使用基于 `location.href` 的绝对路径，确保调试和生产环境都能正确读取。

### 核心修复

1. **新增 `getBasePath()` 方法**：
   - 从 `location.href` 提取基础路径
   - 兼容 `http://` 和 `file://` 协议
   - 去掉文件名和 hash，只保留目录路径

2. **修改模板加载逻辑**：
   - `loadBuiltInTemplates()`：使用 `getBasePath()` 构建绝对路径
   - `initializeTemplates()`：同样使用绝对路径
   - `getTemplateFilePath()`（在 `TemplateManager.vue` 中）：使用绝对路径

### 代码示例

```javascript
/**
 * 获取当前页面的基础路径（兼容调试和生产环境）
 */
getBasePath() {
  try {
    const url = new URL(window.location.href)
    const pathname = url.pathname.replace(/\/[^/]+$/, '') || '/'
    
    // 处理 file:// 协议（host 为空）
    if (url.protocol === 'file:') {
      return `${url.protocol}//${pathname}/`
    }
    
    // 处理 http/https 协议
    return `${url.protocol}//${url.host}${pathname}/`
  } catch (error) {
    // 备用方法
    const href = window.location.href
    const base = href.replace(/\/[^/]+$/, '/').replace(/#.*$/, '')
    return base
  }
}
```

### 路径构建

```javascript
// 之前（错误）：使用相对路径
const configUrl = `${baseUrl}templates/templates.json`

// 现在（正确）：使用基于 location.href 的绝对路径
const basePath = this.getBasePath()
const configUrl = `${basePath}templates/templates.json`
```

## 修改的文件

1. **`src/utils/templateManager.js`**
   - 新增 `getBasePath()` 方法
   - 修改 `loadBuiltInTemplates()` 使用绝对路径
   - 修改 `initializeTemplates()` 使用绝对路径

2. **`src/views/TemplateManager.vue`**
   - 新增 `getBasePath()` 函数
   - 修改 `getTemplateFilePath()` 使用绝对路径

## 验证方法

1. **调试环境**：
   - 打开开发者工具（F12）
   - 查看控制台日志，确认 `basePath` 为 `http://127.0.0.1:3889/`
   - 确认模板能正常加载

2. **生产环境**：
   - 打包插件并安装
   - 打开开发者工具（F12）
   - 查看控制台日志，确认 `basePath` 为 `file:///C:/ProgramData/.../`
   - 确认模板能正常加载

## 注意事项

1. **public 目录**：确保 `public/templates` 目录存在，打包时会原样拷贝
2. **路径格式**：`file://` 协议下，`URL.host` 为空，需要特殊处理
3. **错误处理**：添加了 try-catch 和备用方法，确保 URL 解析失败时也能正常工作
