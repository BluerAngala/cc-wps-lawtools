# 模板文档创建指南

## 📋 概述

本指南帮助你创建 WPS 插件所需的 Word 模板文档。

## 🚀 快速开始

### 方法一：自动创建（推荐）⚡

使用 Node.js 脚本一键创建所有模板：

```bash
# 在项目根目录运行
pnpm create-templates

# 或者
npm run create-templates
```

脚本会自动：
- ✅ 创建 5 个 Word 文档（.docx 格式）
- ✅ 设置标准格式和样式
- ✅ 保存到 public/templates/ 目录
- ✅ 变量占位符用红色标注

### 方法二：手动创建 📝

1. 打开 `public/templates/模板内容-请复制到Word.txt`
2. 复制每个模板的内容到 Word
3. 设置格式（见下方格式规范）
4. 保存为对应的 .docx 文件

## 📁 需要创建的文件

在 `public/templates/` 目录下创建以下文件：

```
public/templates/
├── 民事起诉状.docx     ✅ 诉讼文书
├── 民事答辩状.docx     ✅ 诉讼文书
├── 代理词.docx         ✅ 诉讼文书
├── 劳动合同.docx       ✅ 合同模板
└── 律师函.docx         ✅ 律师函
```

## 🎨 格式规范

### 页面设置
- **纸张**：A4 (210mm × 297mm)
- **页边距**：
  - 上：2.54cm
  - 下：2.54cm
  - 左：3.17cm
  - 右：3.17cm

### 字体设置
| 元素 | 字体 | 字号 | 样式 | 对齐 |
|------|------|------|------|------|
| 文档标题 | 黑体 | 二号(22磅) | 加粗 | 居中 |
| 一级标题 | 黑体 | 小三(15磅) | 加粗 | 左对齐 |
| 二级标题 | 黑体 | 四号(14磅) | 加粗 | 左对齐 |
| 正文 | 宋体 | 小四(12磅) | 常规 | 两端对齐 |
| 变量占位符 | 宋体 | 小四(12磅) | 红色/蓝色 | - |

### 段落设置
- **首行缩进**：2字符
- **行距**：1.5倍
- **段前间距**：0.5行
- **段后间距**：0.5行

## 🔤 变量占位符

模板中使用 `{{变量名}}` 格式，建议设置为红色或蓝色以便识别。

### 常用变量列表

**人员信息**
- `{{原告姓名}}`、`{{被告姓名}}`、`{{答辩人姓名}}`
- `{{性别}}`、`{{出生日期}}`、`{{民族}}`
- `{{住址}}`、`{{联系电话}}`、`{{身份证号}}`

**单位信息**
- `{{公司名称}}`、`{{公司地址}}`、`{{公司电话}}`
- `{{法定代表人}}`
- `{{律师事务所名称}}`、`{{律师姓名}}`、`{{执业证号}}`

**案件信息**
- `{{法院名称}}`、`{{案由}}`
- `{{诉讼请求1}}`、`{{诉讼请求2}}`
- `{{事实与理由}}`、`{{答辩意见}}`

**其他**
- `{{日期}}`、`{{期限}}`
- `{{工资金额}}`、`{{岗位名称}}`

## ✅ 验证清单

创建完成后，请检查：

- [ ] 文件名完全匹配配置文件中的 `fileName`
- [ ] 文件格式为 `.docx`（不是 .doc）
- [ ] 文件保存在 `public/templates/` 目录
- [ ] 包含所有必要的变量占位符
- [ ] 格式美观、专业
- [ ] 变量占位符用颜色标注（可选）
- [ ] 文档可以正常打开

## 🧪 测试

1. 创建完文档后，刷新 WPS 插件
2. 进入"文档模板管理"页面
3. 检查模板是否正常显示
4. 尝试"新建文档"功能
5. 尝试"应用到当前文档"功能

## 🔧 故障排除

### 问题：模板不显示
- 检查文件名是否与 `templates.json` 中的 `fileName` 完全一致
- 检查文件是否在正确的目录
- 刷新页面重新加载

### 问题：应用模板失败
- 确保在 WPS 环境中运行
- 检查文件是否为有效的 .docx 格式
- 查看浏览器控制台的错误信息

### 问题：格式丢失
- 新建文档时格式会保留
- 应用到当前文档时只保留文本内容
