# 路径管理统一说明

## 概述

本次更新统一管理了项目中所有的文件保存路径，使用 `pathManager` 统一管理配置文件、模板文件和缓存文件的存储位置。

## 新的路径结构

### 基础路径

项目使用 WPS 应用数据目录作为基础路径：

```
{Application.Env.GetAppDataPath()}/kingsoft/wps/jsaddons/
```

### 重要说明

⚠️ **jsaddons 整个目录都是只读的**：WPS 在运行时会锁定 `kingsoft/wps/jsaddons` 整个目录，包括其下的所有子目录，无法创建任何新目录或文件。因此，配置文件目录必须放在 **AppDataPath 根目录下**。

### 目录结构

```
{AppDataPath}/                                    # C:/Users/xxx/AppData/Roaming
├── wps_lawtools_1.0.0_config/                   # 配置目录（可写，在 AppDataPath 根目录）
│   ├── config.json                               # 应用配置文件
│   ├── templates/                                # 用户模板目录
│   │   ├── templates.json                        # 模板配置文件
│   │   └── *.docx                                # 用户保存的模板文件
│   ├── cache/                                    # AI 缓存目录
│   │   └── *.json                                # 缓存文件
│   └── logs/                                     # 日志目录
│       └── *.log                                 # 日志文件
└── kingsoft/
    └── wps/
        └── jsaddons/
            └── wps_lawtools_1.0.0/               # 项目根目录（只读，由 WPS 管理）
                └── public/
                    └── templates/                 # 内置模板目录（只读）
                        ├── templates.json
                        └── *.docx
```

## 核心组件

### 1. PathManager（路径管理器）

**文件位置：** `src/utils/pathManager.js`

**主要功能：**
- 统一管理所有文件路径
- 自动创建必需的目录
- 提供路径信息调试功能

**主要方法：**

```javascript
// 获取项目根目录（只读）
pathManager.getProjectRoot()
// 返回: {AppDataPath}/kingsoft/wps/jsaddons/wps_lawtools_1.0.0

// 获取配置目录（可写，在 AppDataPath 根目录）
pathManager.getConfigDir()
// 返回: {AppDataPath}/wps_lawtools_1.0.0_config

// 获取模板目录（用户模板）
pathManager.getTemplatesDir()
// 返回: {ConfigDir}/templates

// 获取缓存目录
pathManager.getCacheDir()
// 返回: {ConfigDir}/cache

// 获取日志目录
pathManager.getLogsDir()
// 返回: {ConfigDir}/logs

// 获取配置文件路径
pathManager.getConfigFilePath()
// 返回: {ConfigDir}/config.json

// 获取模板配置文件路径
pathManager.getTemplatesConfigPath()
// 返回: {ConfigDir}/templates/templates.json

// 获取内置模板相对路径（用于 HTTP 请求）
pathManager.getBuiltInTemplatesPath()
// 返回: /templates

// 获取内置模板实际路径（用于文件系统访问）
pathManager.getBuiltInTemplatesRealPath()
// 返回: {ProjectRoot}/public/templates

// 确保目录存在
pathManager.ensureDir(dirPath)

// 确保所有必需目录存在
pathManager.ensureAllDirs()

// 获取路径信息（用于调试）
pathManager.getPathInfo()
```

## 初始化

在应用启动时（`src/main.js`），会自动初始化所有必需的目录：

```javascript
import { pathManager } from './utils/pathManager.js'
if (typeof window !== 'undefined' && window.Application) {
  setTimeout(() => {
    pathManager.ensureAllDirs()
  }, 1000)
}
```

## 路径版本管理

项目路径包含版本号（`wps_lawtools_1.0.0`），便于：
1. 不同版本的配置互不干扰
2. 版本升级时可以保留旧版本配置
3. 方便进行版本迁移

## 内置模板与用户模板

### 内置模板
- 位置：`public/templates/` （项目资源目录）
- 通过 HTTP 请求访问
- 只读，不可修改

### 用户保存的模板
- 位置：`{ProjectRoot}/config/templates/` （用户数据目录）
- 通过 WPS FileSystem API 访问
- 可读写

## 优势

1. **统一管理**：所有路径通过 `pathManager` 统一管理，便于维护
2. **版本隔离**：不同版本使用不同目录，避免冲突
3. **自动创建**：应用启动时自动创建所有必需目录
4. **易于调试**：提供完整的路径信息查询功能
5. **符合规范**：使用 WPS 推荐的应用数据目录结构

## 注意事项

1. 路径中的版本号需要与 `package.json` 中的版本号保持一致
2. 所有路径统一使用正斜杠（`/`），自动处理 Windows 反斜杠
3. 内置模板使用相对路径，用户模板使用绝对路径
4. 目录创建采用递归方式，自动创建父目录
