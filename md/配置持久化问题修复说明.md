# 配置持久化问题修复说明

## 问题描述

用户在合同审查功能中新建关键词批注配置后，重启 WPS 后发现配置丢失，仍然使用默认配置。

## 问题分析

### 根本原因

配置虽然保存到了文件，但组件重新加载时没有从配置文件读取，而是使用了硬编码的默认值。

### 问题流程

```
1. 用户修改配置 → 触发 updateConfig
2. updateConfig → 保存到 configs.value（内存）
3. 用户点击"保存配置" → 调用 contractService.saveConfig()
4. 配置写入文件 ✅

--- 重启 WPS ---

5. loadConfig() 读取配置 ✅
6. 但是 SmartCommenter 组件使用硬编码的默认值 ❌
7. 用户看到的是默认配置而不是保存的配置 ❌
```

### 代码问题点

#### 1. `SmartCommenter.vue` 使用硬编码默认值

```javascript
// ❌ 问题代码
const configForm = reactive({
  keywordList: [
    { keyword: '第一条', comment: '...', actionType: '批注' },
    // ... 硬编码的默认值
  ],
  reviewKeywordList: [
    // ... 硬编码的默认值
  ]
})
```

#### 2. 没有从 props 接收配置

```vue
<!-- ❌ 问题代码 -->
<SmartCommenter
  :processing="..."
  @execute="..."
  @update-config="..."
/>
<!-- 缺少配置传递 -->
```

#### 3. 配置保存逻辑不完整

```javascript
// ❌ 问题代码
const updateConfig = (type, config) => {
  configs.value[type] = config
  contractService.saveConfig(configs.value) // 只保存部分配置
}
```

## 修复方案

### 1. 传递配置到组件

**文件：`src/views/ContractServices.vue`**

```vue
<!-- ✅ 修复后 -->
<SmartCommenter
  :processing="..."
  :keyword-config="configs.keyword"
  :review-config="configs.review"
  @execute="..."
  @update-config="..."
/>
```

### 2. 组件接收并监听配置

**文件：`src/components/SmartCommenter.vue`**

```javascript
// ✅ 添加 props
const props = defineProps({
  processing: { type: Boolean, default: false },
  keywordConfig: { type: Object, default: () => ({}) },
  reviewConfig: { type: Object, default: () => ({}) }
})

// ✅ 监听 props 变化，同步到内部状态
watch(
  () => props.keywordConfig,
  (newConfig) => {
    if (newConfig && newConfig.keywordList && newConfig.keywordList.length > 0) {
      configForm.keywordList = newConfig.keywordList
    }
  },
  { immediate: true, deep: true }
)

watch(
  () => props.reviewConfig,
  (newConfig) => {
    if (newConfig && newConfig.contractReviewRules && newConfig.contractReviewRules.length > 0) {
      configForm.reviewKeywordList = newConfig.contractReviewRules.map(rule => ({
        keyword: rule.reviewRules,
        comment: rule.reviewRequirements,
        actionType: rule.actionType
      }))
    }
  },
  { immediate: true, deep: true }
)
```

### 3. 完善配置保存逻辑

**文件：`src/views/ContractServices.vue`**

```javascript
// ✅ 统一的配置更新方法
const updateConfig = (type, config) => {
  configs.value[type] = config
  // 立即保存到 appConfig
  saveConfigToAppConfig()
}

// ✅ 保存配置到 appConfig（保存完整配置）
const saveConfigToAppConfig = () => {
  const allConfig = appConfig.getConfig()
  
  // 更新对应的配置项
  if (configs.value.extractor) {
    allConfig.extractor = configs.value.extractor
  }
  if (configs.value.keyword) {
    allConfig.keyword = configs.value.keyword
  }
  if (configs.value.review) {
    allConfig.review = configs.value.review
  }
  
  appConfig.saveConfig(allConfig)
}

// ✅ 处理 SmartCommenter 的配置更新
const updateSmartConfig = (configForm) => {
  // 保存关键词配置
  if (configForm.keywordList) {
    updateConfig('keyword', { keywordList: configForm.keywordList })
  }
  
  // 保存审查配置（转换为标准格式）
  if (configForm.reviewKeywordList) {
    const contractReviewRules = configForm.reviewKeywordList.map(item => ({
      reviewRules: item.keyword,
      reviewRequirements: item.comment,
      actionType: item.actionType
    }))
    updateConfig('review', { contractReviewRules })
  }
}
```

## 修复后的流程

```
1. 用户修改配置 → 触发 updateConfig
2. updateConfig → 更新 configs.value（内存）
3. updateConfig → 自动调用 saveConfigToAppConfig()
4. saveConfigToAppConfig → 合并完整配置并保存到文件 ✅

--- 重启 WPS ---

5. loadConfig() → 从文件读取完整配置 ✅
6. configs.value = { keyword: {...}, review: {...} }
7. 传递给 SmartCommenter 组件
   :keyword-config="configs.keyword"
   :review-config="configs.review"
8. SmartCommenter 的 watch 监听到 props 变化 ✅
9. 自动同步到 configForm ✅
10. 用户看到保存的配置 ✅
```

## 优化点

1. **自动保存** - 配置修改后立即保存，无需点击"保存配置"按钮
2. **静默保存** - 配置修改时静默保存，不显示提示，避免频繁打扰用户
3. **数据同步** - 使用 Vue 的 watch 机制自动同步配置
4. **格式转换** - 自动处理内部格式和存储格式的转换
5. **完整保存** - 保存时合并完整的 appConfig，不会丢失其他配置

## 注意事项

1. **配置格式** - 不同模块使用不同的字段名：
   - 关键词模式：`keywordList`
   - AI预审模式：`contractReviewRules`

2. **静默保存** - 配置修改时自动静默保存，不会显示提示消息
   - 只有点击"保存配置"按钮时才显示"配置已保存"提示
   - 避免频繁提示打扰用户体验

3. **保存按钮** - 虽然配置会自动保存，但仍保留"保存配置"按钮：
   - 用于用户主动确认保存
   - 显示明确的成功/失败提示

4. **配置位置** - 配置保存在用户主目录：
   - Windows: `C:\Users\{用户名}\wps_addon_config\config.json`
   - Linux: `/home/{用户名}/wps_addon_config/config.json`
   - Mac: `/Users/{用户名}/wps_addon_config/config.json`

## 代码实现细节

### 静默保存实现

```javascript
// saveConfigToAppConfig 接受一个参数控制是否显示提示
const saveConfigToAppConfig = (showMessage = true) => {
  const allConfig = appConfig.getConfig()
  
  // 更新配置...
  const success = appConfig.saveConfig(allConfig)
  
  // 只在需要时显示消息
  if (showMessage) {
    if (success) {
      window.$message?.success('配置已保存')
    } else {
      window.$message?.error('保存配置失败')
    }
  }
  
  return success
}

// 自动保存时不显示提示
const updateConfig = (type, config) => {
  configs.value[type] = config
  saveConfigToAppConfig(false) // 静默保存
}

// 手动保存时显示提示
const saveConfig = () => {
  saveConfigToAppConfig(true) // 显示提示
}
```
