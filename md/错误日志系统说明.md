# 错误日志系统说明

## 概述
为了提高用户体验和便于问题追踪，我们将项目中所有的 `alert()` 弹窗改为了统一的错误日志系统。

## 主要改进

### 1. 创建了错误日志工具 (`src/utils/errorLogger.js`)

**功能特性：**
- ✅ 自动记录错误到本地存储（localStorage）
- ✅ 保留最近 100 条错误日志
- ✅ 显示友好的 NaiveUI 消息提示
- ✅ 同时输出到浏览器控制台
- ✅ 支持导出日志为文本文件
- ✅ 支持清空日志
- ✅ 持久化存储，刷新不丢失

**核心方法：**
```javascript
// 记录错误日志
errorLogger.log(message, extra)

// 获取所有日志
errorLogger.getLogs()

// 清空日志
errorLogger.clearLogs()

// 导出日志为文本
errorLogger.exportLogsAsText()

// 下载日志文件
errorLogger.downloadLogs()
```

### 3. 错误日志格式

每条日志包含以下信息：
```javascript
{
  timestamp: "2024-11-14T12:34:56.789Z",  // 时间戳
  message: "错误消息",                     // 错误描述
  method: "methodName",                    // 发生错误的方法名
  url: "...",                              // 相关URL（如适用）
  key: "...",                              // 相关键名（如适用）
  taskId: "...",                           // 任务ID（如适用）
  error: "详细错误信息"                    // 原始错误消息
}
```

## 使用方式

### 在其他文件中使用

```javascript
import errorLogger from '@/utils/errorLogger'

// 记录简单错误
errorLogger.log('操作失败')

// 记录详细错误
errorLogger.log('创建任务失败', {
  method: 'createTask',
  taskId: '123',
  error: error.message
})
```

### 查看错误日志

在浏览器控制台中：
```javascript
// 查看所有日志
errorLogger.getLogs()

// 导出日志文本
console.log(errorLogger.exportLogsAsText())

// 下载日志文件
errorLogger.downloadLogs()

// 清空日志
errorLogger.clearLogs()
```

## 优势对比

### 使用 alert 的问题：
- ❌ 阻塞用户操作
- ❌ 无法追踪历史错误
- ❌ 不够美观
- ❌ 无法批量查看
- ❌ 调试困难

### 使用错误日志的优势：
- ✅ 不阻塞用户操作
- ✅ 自动保存历史记录
- ✅ UI 提示更美观
- ✅ 可批量查看和导出
- ✅ 便于问题追踪和调试
- ✅ 支持详细的上下文信息

## 注意事项

1. **存储限制**：localStorage 有大小限制（通常 5-10MB），因此只保留最近 100 条日志
2. **隐私保护**：导出日志前请检查是否包含敏感信息
3. **消息提示**：需要确保 `window.$message` 已正确挂载（由 NaiveUI 提供）
