# 首次加载欢迎页面功能说明

## 问题描述
之前每次打开 WPS 加载项都会显示欢迎页面，这对于老用户来说体验不好。

## 解决方案
实现了首次加载检测机制，只在用户首次安装使用时显示欢迎页面。

## 实现原理

### 1. 配置管理 (appConfig.js)
使用 WPS FileSystem API 持久化存储配置信息：
- 配置存储在用户主目录下的 `wps_addon_config` 文件夹
- 配置文件：`config.json`
- 关键字段：`system.firstLoadCompleted` - 标记是否已完成首次加载

### 2. 首次加载检测 (ribbon.js)
在 `OnAddinLoad()` 函数中实现检测逻辑：
```javascript
function OnAddinLoad() {
  setTimeout(() => {
    const isFirstLoad = appConfig.isFirstLoad()
    
    if (isFirstLoad) {
      // 首次加载：显示欢迎页面
      showWelcomeDialog()
      // 标记已完成首次加载
      appConfig.markFirstLoadCompleted()
    } else {
      // 非首次加载：跳过欢迎页面
      console.log('非首次加载，跳过欢迎页面')
    }
  }, 1000)
}
```

## 核心方法

### isFirstLoad()
检查是否为首次加载：
```javascript
isFirstLoad() {
  const config = this.getConfig()
  return !config.system?.firstLoadCompleted
}
```

### markFirstLoadCompleted()
标记首次加载完成：
```javascript
markFirstLoadCompleted() {
  const config = this.getConfig()
  config.system.firstLoadCompleted = true
  return this.saveConfig(config)
}
```

### resetFirstLoad()
重置首次加载状态（用于测试或重新演示）：
```javascript
resetFirstLoad() {
  const config = this.getConfig()
  config.system.firstLoadCompleted = false
  return this.saveConfig(config)
}
```

## 使用说明

### 正常使用
- **首次安装**：打开 WPS 时会自动显示欢迎页面
- **后续使用**：不会再显示欢迎页面

### 手动触发
用户可以通过以下方式手动查看欢迎页面：
1. 点击 WPS 功能区的"欢迎页面"按钮（btnShowWelcome）
2. 使用"重置首次加载"按钮（btnResetFirstLoad）后重新启动 WPS

## 配置存储位置
- **Windows**: `C:\Users\[用户名]\wps_addon_config\config.json`
- **Linux**: `~/wps_addon_config/config.json`
- **Mac**: `~/wps_addon_config/config.json`

## 注意事项
1. 首次加载检测依赖 WPS FileSystem API，需要在 WPS 环境中运行
2. 配置文件会自动创建和管理，用户无需手动操作
3. 延迟 1 秒执行检测，确保 WPS 环境完全初始化
