# 文档模板管理功能说明

## 功能概述

文档模板管理功能支持管理和应用常用的法律文书模板，包括诉讼文书、合同模板、律师函等。

## 核心特性

### 1. 模板文件化管理
- ✅ 支持标准 `.docx` 格式的 Word 文档
- ✅ 保留完整的文档格式、样式、表格等
- ✅ 支持变量占位符（如 `{{原告姓名}}`）

### 2. 配置文件驱动
- ✅ 使用 `templates.json` 统一管理模板信息
- ✅ 包含模板名称、分类、描述、使用场景等元数据
- ✅ 易于扩展和维护

### 3. 智能加载机制
- ✅ 启动时只加载配置，不加载文件内容
- ✅ 模板内容按需加载，提升性能
- ✅ 自动适配开发环境和生产环境路径

### 4. 模板应用方式
- ✅ **新建文档**：直接打开模板文件，保留所有格式
- ✅ **应用到当前文档**：将模板内容插入当前文档

### 5. 模板更新功能
- ✅ 支持编辑和更新现有自定义模板
- ✅ 内容预览功能，选择模板时显示内容
- ✅ 区分内置模板和自定义模板

## 配置文件格式

```json
{
  "templates": [
    {
      "id": "template_001",
      "name": "民事起诉状",
      "category": "诉讼文书",
      "description": "适用于一般民事案件的起诉状模板",
      "scene": "民事纠纷、合同纠纷、侵权纠纷等民事案件",
      "fileName": "民事起诉状.docx",
      "isBuiltIn": true
    }
  ]
}
```

## 技术实现

### 路径处理
```javascript
const getTemplateFilePath = (fileName) => {
  const baseUrl = window.location.origin
  return `${baseUrl}/templates/${fileName}`
}
```

### 模板加载
```javascript
// 使用 WPS API 打开 docx 文件
const tempDoc = window.Application.Documents.Open(filePath, false, true)
const content = tempDoc.Range().Text
tempDoc.Close(false)
```

### 应用模板
- **内置模板**：直接使用 WPS API 打开 docx 文件
- **自定义模板**：使用保存的文本内容

## 使用流程

### 添加新模板

1. **创建 Word 文档**
   - 在 Word 中创建模板文档
   - 使用 `{{变量名}}` 作为占位符
   - 保存为 `.docx` 格式

2. **放置文件**
   - 将文档放入 `public/templates/` 目录

3. **更新配置**
   - 编辑 `templates.json`
   - 添加新模板的配置信息

4. **刷新页面**
   - 重新加载页面即可看到新模板

### 应用模板

1. **选择分类**：在标签页中选择对应分类
2. **选择操作**：
   - 点击"应用到当前文档"：将模板应用到当前打开的文档
   - 点击"新建文档"：在新文档中打开模板
3. **编辑内容**：替换模板中的占位符变量

### 管理自定义模板

1. **保存模板**：
   - 点击"保存当前文档为模板"
   - 填写模板信息
   - 选择分类

2. **更新模板**：
   - 点击模板卡片上的"编辑"按钮
   - 或在保存对话框中选择现有模板
   - 修改后点击"更新"

3. **删除模板**：
   - 点击自定义模板的"删除"按钮
   - 确认删除

## 环境适配

### 开发环境
- 从 `http://localhost:3889/templates/` 加载模板
- 支持热更新，修改配置后刷新即可

### 生产环境
- 从打包后的相对路径加载模板
- 确保模板文件包含在构建输出中

## 注意事项

1. **文件格式**：必须使用 `.docx` 格式
2. **文件名一致性**：配置文件中的 `fileName` 必须与实际文件名完全一致
3. **WPS 环境**：模板功能需要在 WPS 环境中运行
4. **格式保留**：新建文档时会保留所有格式，应用到当前文档时只保留文本内容
5. **路径处理**：使用相对路径，自动适配不同环境
