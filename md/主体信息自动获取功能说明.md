# 主体信息自动获取功能说明

## 功能概述

在抽取合同信息后，系统会自动调用 Coze API 获取甲方和乙方的主体信息，并将这些信息添加到抽取结果中。在提交数据到金山文档时，主体信息也会一并提交。

## 核心功能

### 1. 特定主体映射

对于某些特定主体（如政府机关），无需调用 API，直接使用预定义的信息映射。

**配置位置：** `src/services/ai/coze.js`

```javascript
const specialSubjects = {
  '中共广东省委宣传部': {
    info: '广东省委宣传部是广东省委主管意识形态的综合职能部门，位于广东省广州市越秀区合群三马路26号省委大院。统一社会信用代码为11440000006939561H。'
  },
  '中共广东省委宣传部机关工会委员会': {
    info: '广东省委宣传部是依法设立的工会，位于广东省广州市越秀区合群三马路26号省委大院。统一社会信用代码为81440000742971228Y。'
  }
  // 可以继续添加更多特定主体
}
```

**如何添加新的特定主体：**

在 `specialSubjects` 对象中添加新条目，格式如下：

```javascript
'主体名称': {
  info: '主体的详细信息'
}
```

### 2. 自动调用 Coze API

对于非特定主体，系统会自动调用 Coze API 查询企业信息，包括：

- 企业名称
- 统一社会信用代码
- 法定代表人
- 企业类型
- 地址
- 成立日期

### 3. 数据流程

```
用户点击"开始抽取"
    ↓
AI 抽取合同信息（甲方、乙方等）
    ↓
自动获取主体信息
    ├─ 检查是否为特定主体 → 使用映射
    └─ 非特定主体 → 调用 Coze API
    ↓
将主体信息添加到抽取结果
    ↓
用户编辑确认
    ↓
提交到金山文档（包含主体信息）
```

## 实现细节

### 1. 核心函数

**`fetchCompanyInfo(companyName)`**
- 位置：`src/services/ai/coze.js`
- 功能：调用 Coze API 获取企业信息（纯 API 调用，不包含业务逻辑）
- 使用 axios 进行 HTTP 请求
- 返回标准化的企业信息对象

**`fetchSubjectInfo(subjectName, subjectType)`**
- 位置：`src/services/contract/dataSubmitter.js`
- 功能：获取单个主体信息（包含特定主体映射和格式化逻辑）
- 优先检查特定主体映射
- 否则调用 `fetchCompanyInfo` API
- 返回格式化的信息字符串

**`formatCompanyInfo(companyInfo)`**
- 位置：`src/services/contract/dataSubmitter.js`
- 功能：将企业信息对象格式化为可读字符串

### 2. 数据处理流程

**`processExtractedData(rawResult)` - 异步方法**
- 位置：`src/services/contract/dataSubmitter.js`
- 功能：处理抽取的合同数据，自动添加主体信息
- 修改：改为异步方法，在标准化字段后自动获取主体信息

### 3. 字段标准化

主体信息字段已添加到标准字段列表中：

```javascript
const requiredFields = [
  '合同名称', 
  '甲方', 
  '乙方', 
  '其他方', 
  '合同金额', 
  '对接客户', 
  '甲方主体信息',  // 新增
  '乙方主体信息'   // 新增
]
```

## 用户体验

### 1. 提示信息

- 开始获取：显示"正在获取主体信息..."
- 获取成功：显示"主体信息获取成功"
- 获取失败：显示"主体信息获取失败，请手动填写"

### 2. 错误处理

- 如果特定主体获取失败，会记录错误但不影响整体流程
- 如果 API 调用失败，会添加空字段，允许用户手动填写
- 所有错误都会在控制台输出详细日志

### 3. 数据编辑

用户可以在提交前编辑主体信息字段，确保信息准确无误。

## 提交到金山文档

在提交数据时，`甲方主体信息` 和 `乙方主体信息` 字段会与其他合同信息一起提交到金山文档。

## 技术说明

### 1. 异步处理

整个主体信息获取过程是异步的，不会阻塞用户界面。调用链如下：

```
ContractServices.vue (executeExtraction)
    ↓
contractService.processExtractedData() [async]
    ↓
dataSubmitter.processExtractedData() [async]
    ↓
fetchSubjectsInfo() [async]
```

### 2. Coze API 配置

- API 密钥：已在 `coze.js` 中配置
- 工作流 ID：7550481844523221034
- API 端点：https://api.coze.cn/v1/workflow/run
- 超时时间：180秒

### 3. 信息格式化

企业信息会按照以下格式组织：

```
名称：xxx，统一社会信用代码：xxx，法定代表人：xxx，企业类型：xxx，地址：xxx，成立日期：xxx
```

## 维护和扩展

### 添加新的特定主体

编辑 `src/services/contract/dataSubmitter.js`，在文件顶部的 `specialSubjects` 对象中添加新条目。

### 修改信息格式

编辑 `dataSubmitter.js` 中的 `formatCompanyInfo` 方法来调整信息的组织和显示格式。

### 调整字段顺序

在 `normalizeFields` 方法中调整 `requiredFields` 数组的顺序。

## 测试建议

1. **测试特定主体**：使用"中共广东省委宣传部"作为甲方或乙方
2. **测试普通企业**：使用正常的企业名称
3. **测试错误情况**：使用不存在的企业名称
4. **测试编辑功能**：确认可以手动修改主体信息
5. **测试提交功能**：确认主体信息能正确提交到金山文档

## 注意事项

1. Coze API 有调用频率限制，请合理使用
2. 特定主体信息需要定期更新维护
3. 主体信息字段较长，建议在界面上使用多行文本框或折叠显示
4. 如果企业名称不准确，可能导致查询失败，需要用户手动填写

## 架构设计说明

### 职责分离

本功能遵循单一职责原则：

1. **`coze.js`**：纯 API 调用层
   - 仅负责与 Coze API 通信
   - 使用 axios 发送 HTTP 请求
   - 返回标准化的数据结构
   - 不包含任何业务逻辑

2. **`dataSubmitter.js`**：业务逻辑层
   - 管理特定主体映射
   - 处理主体信息获取逻辑
   - 格式化企业信息
   - 协调数据提交流程

这样的设计使得：
- API 调用与业务逻辑分离
- 代码结构清晰，易于维护
- 功能模块职责明确，不混乱
