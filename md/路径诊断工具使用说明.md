# 路径诊断工具使用说明

**文件：** `src/views/PathDiagnostics.vue`  
**访问路径：** `http://127.0.0.1:3889/#/pathdiag`  
**用途：** 诊断和调试文件路径问题

---

## 🎯 功能概述

路径诊断工具帮助你：
1. ✅ 检查 WPS 环境是否可用
2. ✅ 查看所有路径信息
3. ✅ 检查目录是否存在
4. ✅ 测试目录创建功能
5. ✅ 测试文件读写功能

---

## 🚀 使用方法

### 1. 访问诊断页面

在 WPS 中打开：
```
http://127.0.0.1:3889/#/pathdiag
```

或在开发服务器中直接访问。

### 2. 查看环境状态

页面会自动显示：
- Application 对象是否可用
- FileSystem API 是否可用
- Env API 是否可用

### 3. 查看路径信息

显示所有关键路径：
- AppDataPath
- 项目根目录
- 配置目录
- 模板目录
- 缓存目录
- 日志目录

### 4. 检查目录状态

显示每个目录是否存在：
- ✅ 绿色标签 = 目录存在
- ❌ 红色标签 = 目录不存在

### 5. 测试创建目录

点击**"测试创建目录"**按钮：
- 尝试创建所有必需的目录
- 显示每个操作的详细结果
- 包含 Mkdir 返回值和错误信息

### 6. 测试保存文件

点击**"测试保存文件"**按钮：
- 在配置目录创建测试文件
- 验证文件是否成功保存
- 读取文件内容验证
- 自动清理测试文件

## 📊 测试结果解读

### 成功的情况

```json
{
  "timestamp": "2025-11-14T13:44:32.000Z",
  "operations": [
    {
      "name": "配置目录",
      "path": "C:/Users/xxx/AppData/Roaming/wps_lawtools_1.0.0_config",
      "existsBefore": false,
      "mkdirResult": true,
      "existsAfter": true,
      "success": true
    }
  ]
}
```

**说明：**
- `existsBefore: false` - 目录之前不存在
- `mkdirResult: true` - Mkdir 返回成功
- `existsAfter: true` - 创建后目录存在
- `success: true` - 操作成功

### 失败的情况

```json
{
  "timestamp": "2025-11-14T13:44:32.000Z",
  "operations": [
    {
      "name": "配置目录",
      "path": "C:/Users/xxx/AppData/Roaming/kingsoft/wps/jsaddons/xxx",
      "existsBefore": false,
      "mkdirResult": true,
      "existsAfter": false,
      "success": false
    }
  ]
}
```

**问题分析：**
- `mkdirResult: true` - Mkdir 返回成功
- `existsAfter: false` - 但目录实际不存在
- **原因：** 该路径在只读目录下，无法创建

**解决方案：** 修改路径到可写位置

## 🐛 常见问题排查

### 问题 1：目录创建失败

**症状：**
```
[ERROR] 创建目录失败
{
  "dirPath": "C:/Users/xxx/AppData/Roaming/kingsoft/wps/jsaddons/..."
}
```

**原因：** jsaddons 目录是只读的

**解决：** 配置目录应该在 `{AppDataPath}/wps_lawtools_1.0.0_config`，而不是在 jsaddons 下

### 问题 2：文件保存失败

**症状：** WriteFile 返回 false 或文件不存在

**检查步骤：**
1. 确认目录已创建
2. 确认路径不在只读目录
3. 确认路径格式正确（使用正斜杠）
4. 确认有文件系统权限

### 问题 3：路径为 null

**症状：** `pathManager.getConfigDir()` 返回 null

**原因：** WPS 环境未正确初始化

**解决：**
1. 检查是否在 WPS 中运行
2. 等待 WPS 环境完全加载
3. 检查 `Application.Env.GetAppDataPath()` 是否可用

## 🔧 调试技巧

### 在控制台中调试

```javascript
// 导入 pathManager
import { pathManager } from '@/utils/pathManager.js'

// 查看完整路径信息
console.log(pathManager.getPathInfo())

// 测试创建目录
const configDir = pathManager.getConfigDir()
console.log('配置目录:', configDir)
console.log('目录是否存在:', window.Application.FileSystem.Exists(configDir))

// 尝试创建
pathManager.ensureDir(configDir)

// 验证
console.log('创建后是否存在:', window.Application.FileSystem.Exists(configDir))
```

打开浏览器控制台（F12），查看：
- `[INFO]` 级别：正常操作
- `[WARN]` 级别：警告信息
- `[ERROR]` 级别：错误详情

## 📝 最佳实践

### 1. 路径使用规范

```javascript
// ✅ 好的做法 - 使用 pathManager
const templatesDir = pathManager.getTemplatesDir()
if (templatesDir) {
  pathManager.ensureDir(templatesDir)
  const filePath = templatesDir + '/myfile.docx'
  // 使用 filePath...
}

// ❌ 不好的做法 - 硬编码路径
const filePath = 'C:/Users/xxx/AppData/Roaming/...'
```

### 2. 目录创建检查

```javascript
// ✅ 好的做法 - 先确保目录存在
const configDir = pathManager.getConfigDir()
if (!configDir) {
  window.$message?.error('无法获取配置目录')
  return
}

if (!pathManager.ensureDir(configDir)) {
  window.$message?.error('创建目录失败')
  return
}

// 然后再保存文件
fs.WriteFile(configDir + '/file.txt', content)
```

### 3. 错误处理

```javascript
try {
  const templatesDir = pathManager.getTemplatesDir()
  if (!templatesDir) throw new Error('无法获取模板目录')
  
  pathManager.ensureDir(templatesDir)
  
  const filePath = templatesDir + '/template.docx'
  doc.SaveAs2(filePath, 16)
  
  window.$message?.success('模板保存成功')
} catch (error) {
  console.error('保存失败:', error)
  window.$message?.error('保存失败: ' + error.message)
}
```
