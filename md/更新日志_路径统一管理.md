## 📋 更新概述

本次更新实现了项目文件路径的统一管理，将所有配置文件、模板文件和缓存文件统一存储到项目专用目录。

### 新的目录结构

```
{Application.Env.GetAppDataPath()}/
└── kingsoft/wps/jsaddons/
    └── wps_lawtools_1.0.0/          # 项目根目录（项目名_版本号）
        └── config/                   # 配置根目录
            ├── config.json           # 应用配置文件
            ├── templates/            # 模板目录
            │   ├── templates.json    # 模板配置文件
            │   └── *.docx            # 用户保存的模板文件
            └── cache/                # AI 缓存目录
                └── *.json            # 缓存文件
```

## 🆕 新增文件

### 1. PathManager（路径管理器）
**文件：** `src/utils/pathManager.js`

统一管理所有文件路径的核心工具类，提供：
- 获取各类目录路径的方法
- 自动创建目录的功能
- 路径信息查询接口

**主要 API：**
- `getProjectRoot()` - 获取项目根目录
- `getConfigDir()` - 获取配置目录
- `getTemplatesDir()` - 获取模板目录
- `getCacheDir()` - 获取缓存目录
- `getConfigFilePath()` - 获取配置文件路径
- `getTemplatesConfigPath()` - 获取模板配置文件路径
- `getBuiltInTemplatesPath()` - 获取内置模板路径
- `ensureDir(dirPath)` - 确保目录存在
- `ensureAllDirs()` - 确保所有必需目录存在
- `getPathInfo()` - 获取路径信息（调试用）

### 2. 测试页面
**文件：** `test/test-path-manager.html`

用于测试路径管理器的功能页面，包括：
- WPS 环境检查
- 路径信息展示
- 目录和文件状态检查
- 目录创建测试

### 3. 文档
**文件：** `md/路径管理统一说明.md`

详细说明路径管理的实现方式和使用方法。

## ✨ 功能改进

### 1. 路径版本管理
- 项目路径包含版本号（`wps_lawtools_1.0.0`）
- 不同版本的配置互不干扰
- 方便版本升级和配置迁移

### 2. 自动目录创建
- 应用启动时自动创建所有必需目录
- 递归创建父目录，无需手动干预
- 目录创建失败有详细日志记录

### 3. 统一路径管理
- 所有路径通过 `pathManager` 统一管理
- 代码更简洁，易于维护
- 避免路径硬编码，降低出错概率

### 4. 调试支持
- 提供 `getPathInfo()` 方法查看完整路径信息
- 包含目录和文件的存在状态检查
- 方便问题排查和调试

## 🔧 技术细节

### 路径规范
- 所有路径统一使用正斜杠（`/`）
- 自动处理 Windows 反斜杠转换
- 路径末尾斜杠自动清理

### 目录结构
- 项目根目录：`{AppDataPath}/kingsoft/wps/jsaddons/{projectName}_{version}`
- 配置目录：`{ProjectRoot}/config`
- 模板目录：`{ProjectRoot}/config/templates`
- 缓存目录：`{ProjectRoot}/config/cache`

### 模板管理
- 内置模板：`public/templates/`（只读，HTTP 访问）
- 用户模板：`{ProjectRoot}/config/templates/`（读写，FileSystem API 访问）

## ⚠️ 注意事项

1. **版本号管理**
   - 路径中的版本号需与 `package.json` 保持一致
   - 版本升级时会创建新的目录结构

2. **旧数据迁移**
   - 如果之前使用了不同的配置目录，需要手动迁移数据
   - 旧目录路径：`{HomePath}/wps_addon_config`
   - 新目录路径：`{AppDataPath}/kingsoft/wps/jsaddons/wps_lawtools_1.0.0/config`

3. **权限问题**
   - 确保 WPS 有权限访问 AppDataPath
   - 目录创建失败时检查文件系统权限

## 👨‍💻 开发者指南

### 获取路径
```javascript
import { pathManager } from '@/utils/pathManager.js'

// 获取各类目录
const projectRoot = pathManager.getProjectRoot()
const configDir = pathManager.getConfigDir()
const templatesDir = pathManager.getTemplatesDir()
const cacheDir = pathManager.getCacheDir()

// 获取文件路径
const configFile = pathManager.getConfigFilePath()
const templatesConfig = pathManager.getTemplatesConfigPath()
```

### 确保目录存在
```javascript
// 确保单个目录存在
pathManager.ensureDir('/path/to/directory')

// 确保所有必需目录存在
pathManager.ensureAllDirs()
```

### 调试路径信息
```javascript
const pathInfo = pathManager.getPathInfo()
console.log(pathInfo)
```
